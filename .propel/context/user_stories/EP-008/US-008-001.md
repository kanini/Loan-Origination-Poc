# User Story: Document History Page with Search and Filter

**Epic:** EP-008 - Document History & Library Management  
**Story ID:** US-008-001  
**Priority:** P0 - Critical  
**Estimate:** 8 Story Points (3-4 days)

---

## User Story

**As a** Financial Analyst  
**I want** to search and filter my document history with multiple criteria  
**So that** I can quickly find specific documents from my past processing activities

---

## Acceptance Criteria

### AC1: History Page Layout
**Given** I navigate to the History page  
**When** the page loads  
**Then**:
- Page title "Document History" (H1)
- Search bar at top (40% width on desktop, full width on mobile)
- Filter controls: Date Range, Status, Model dropdowns
- View toggle: Grid View | List View
- Document display area (grid or table)
- Pagination controls at bottom
- "Clear Filters" button when filters are active

### AC2: Search Functionality
**Given** I want to find specific documents  
**When** I type in the search bar  
**Then**:
- Search input has placeholder "Search by filename..."
- Search is debounced (300ms delay)
- Results filter in real-time as I type
- Search is case-insensitive
- Search matches partial filenames
- Search icon displayed in input
- Clear button (X) appears when text is entered

### AC3: Date Range Filter
**Given** I want to filter by upload date  
**When** I open the Date Range dropdown  
**Then**:
- Options: "All Time", "Last 7 Days", "Last 30 Days", "Last 90 Days", "Custom"
- Default selection: "All Time"
- Selecting an option filters documents immediately
- "Custom" option opens date picker for start and end dates
- Selected filter shows in dropdown label
- Filter persists during session

### AC4: Status Filter
**Given** I want to filter by processing status  
**When** I open the Status dropdown  
**Then**:
- Options: "All", "Success", "Failed", "Processing"
- Default selection: "All"
- Selecting an option filters documents immediately
- Success count badge shows number of successful documents
- Failed count badge shows number of failed documents
- Processing count shows active processing jobs

### AC5: Model Filter
**Given** I want to filter by LLM model  
**When** I open the Model dropdown  
**Then**:
- Options: "All", "OpenAI GPT", "Google Gemini"
- Default selection: "All"
- Selecting an option filters documents immediately
- Model icons displayed next to options
- Filter shows count of documents per model

### AC6: Grid View Display
**Given** I select Grid View  
**When** documents are displayed  
**Then**:
- Documents shown as cards in grid layout
- 3-4 columns on desktop (≥1024px)
- 2 columns on tablet (768px-1023px)
- 1 column on mobile (<768px)
- Each card shows: thumbnail, filename, upload date, status badge, model badge, actions
- Cards have hover effect (shadow lift)
- Grid has consistent spacing (16px gap)

### AC7: List View Display
**Given** I select List View  
**When** documents are displayed  
**Then**:
- Documents shown in table format
- Columns: Thumbnail, Filename, Upload Date, Status, Model, Actions
- Table headers are sticky when scrolling
- Row striping for readability
- Sortable columns (click header to sort)
- Responsive table with horizontal scroll on mobile

### AC8: Document Card/Row Content
**Given** a document is displayed  
**When** I view the card or row  
**Then**:
- PDF thumbnail (64×64px) or placeholder icon
- Filename truncated to fit (max 40 characters with ellipsis)
- Full filename shown on hover (tooltip)
- Upload date formatted as "MMM DD, YYYY HH:MM"
- Status badge: Green "Success", Red "Failed", Blue "Processing"
- Model badge: "OpenAI GPT" or "Google Gemini"
- Action buttons: "View" (eye icon), "Delete" (trash icon)

### AC9: Document Navigation
**Given** I want to view a document's results  
**When** I click on a document card or "View" button  
**Then**:
- Navigate to Results page for that document (/results/{documentId})
- Document data loads on Results page
- Browser back button returns to History page
- Filters and scroll position preserved when returning

### AC10: Delete Functionality
**Given** I want to delete a document  
**When** I click the Delete button  
**Then**:
- Confirmation dialog appears: "Are you sure you want to delete this document? This action cannot be undone."
- Dialog has "Cancel" and "Delete" buttons
- Clicking "Cancel" closes dialog without action
- Clicking "Delete" removes document from list
- Optimistic update (immediate removal from UI)
- Success toast: "Document deleted"
- Soft delete in database (maintains audit trail)

### AC11: Empty State
**Given** no documents exist  
**When** I view the History page  
**Then**:
- Empty state illustration (document icon)
- Heading: "No documents processed yet"
- Description: "Upload your first PDF document to extract entity data and build your document library."
- Primary CTA: "Upload Your First Document" button
- Button navigates to Upload page

### AC12: Search/Filter Empty State
**Given** my search or filters return no results  
**When** no documents match the criteria  
**Then**:
- Empty state message: "No documents found"
- Description: "No documents match your search criteria. Try adjusting your filters or search term."
- "Clear Filters" button to reset all filters
- Search term and active filters displayed above message

### AC13: Loading State
**Given** the History page is loading data  
**When** documents are being fetched  
**Then**:
- Skeleton screens for document cards/rows (10 skeletons)
- Skeletons match final card/row layout
- Search and filter controls remain interactive
- No blank screen during loading

### AC14: Error State
**Given** the API request fails  
**When** documents cannot be loaded  
**Then**:
- Error icon displayed
- Message: "Failed to load documents. Please try again."
- "Refresh" button to retry data fetch
- Filters remain in current state
- Error is logged for debugging

## Technical Details

### Component Implementation
```typescript
// src/pages/History.tsx
import { useState, useMemo } from 'react'
import { useQuery } from '@tanstack/react-query'
import { Search, Grid, List, Filter } from 'lucide-react'
import { Input } from '@/components/ui/input'
import { Select } from '@/components/ui/select'
import { Button } from '@/components/ui/button'
import { Toggle } from '@/components/ui/toggle'
import { DocumentGrid } from '@/components/history/DocumentGrid'
import { DocumentTable } from '@/components/history/DocumentTable'
import { Pagination } from '@/components/common/Pagination'
import { useDebounce } from '@/hooks/useDebounce'
import { fetchDocuments } from '@/api/documents'

type ViewMode = 'grid' | 'list'
type DateRange = 'all' | '7days' | '30days' | '90days' | 'custom'
type Status = 'all' | 'success' | 'failed' | 'processing'
type Model = 'all' | 'openai' | 'gemini'

export default function History() {
  const [viewMode, setViewMode] = useState<ViewMode>('grid')
  const [searchTerm, setSearchTerm] = useState('')
  const [dateRange, setDateRange] = useState<DateRange>('all')
  const [status, setStatus] = useState<Status>('all')
  const [model, setModel] = useState<Model>('all')
  const [page, setPage] = useState(1)
  const [itemsPerPage, setItemsPerPage] = useState(20)

  const debouncedSearch = useDebounce(searchTerm, 300)

  const { data, isLoading, error } = useQuery({
    queryKey: ['documents', debouncedSearch, dateRange, status, model, page, itemsPerPage],
    queryFn: () => fetchDocuments({
      search: debouncedSearch,
      dateRange,
      status: status !== 'all' ? status : undefined,
      model: model !== 'all' ? model : undefined,
      page,
      limit: itemsPerPage,
    }),
  })

  const hasActiveFilters = useMemo(() => {
    return searchTerm || dateRange !== 'all' || status !== 'all' || model !== 'all'
  }, [searchTerm, dateRange, status, model])

  const clearFilters = () => {
    setSearchTerm('')
    setDateRange('all')
    setStatus('all')
    setModel('all')
    setPage(1)
  }

  if (isLoading) return <HistorySkeleton />
  if (error) return <HistoryError onRetry={() => window.location.reload()} />
  if (!data?.documents || data.documents.length === 0) {
    return hasActiveFilters ? (
      <SearchEmptyState onClearFilters={clearFilters} />
    ) : (
      <HistoryEmpty />
    )
  }

  return (
    <div className="max-w-screen-2xl mx-auto px-4 py-8">
      <h1 className="text-h1 font-bold mb-6">Document History</h1>

      {/* Search and Filters */}
      <div className="flex flex-col md:flex-row gap-4 mb-6">
        {/* Search */}
        <div className="flex-1">
          <Input
            type="search"
            placeholder="Search by filename..."
            value={searchTerm}
            onChange={(e) => setSearchTerm(e.target.value)}
            icon={<Search className="h-4 w-4" />}
          />
        </div>

        {/* Filters */}
        <div className="flex gap-2">
          <Select value={dateRange} onValueChange={setDateRange}>
            <option value="all">All Time</option>
            <option value="7days">Last 7 Days</option>
            <option value="30days">Last 30 Days</option>
            <option value="90days">Last 90 Days</option>
          </Select>

          <Select value={status} onValueChange={setStatus}>
            <option value="all">All Status</option>
            <option value="success">Success</option>
            <option value="failed">Failed</option>
            <option value="processing">Processing</option>
          </Select>

          <Select value={model} onValueChange={setModel}>
            <option value="all">All Models</option>
            <option value="openai">OpenAI GPT</option>
            <option value="gemini">Google Gemini</option>
          </Select>

          {/* View Toggle */}
          <div className="flex gap-1 border rounded-md p-1">
            <Toggle
              pressed={viewMode === 'grid'}
              onPressedChange={() => setViewMode('grid')}
            >
              <Grid className="h-4 w-4" />
            </Toggle>
            <Toggle
              pressed={viewMode === 'list'}
              onPressedChange={() => setViewMode('list')}
            >
              <List className="h-4 w-4" />
            </Toggle>
          </div>
        </div>
      </div>

      {/* Clear Filters */}
      {hasActiveFilters && (
        <div className="mb-4">
          <Button variant="tertiary" onClick={clearFilters}>
            Clear Filters
          </Button>
        </div>
      )}

      {/* Document Display */}
      {viewMode === 'grid' ? (
        <DocumentGrid documents={data.documents} />
      ) : (
        <DocumentTable documents={data.documents} />
      )}

      {/* Pagination */}
      <Pagination
        currentPage={page}
        totalPages={data.totalPages}
        itemsPerPage={itemsPerPage}
        totalItems={data.totalCount}
        onPageChange={setPage}
        onItemsPerPageChange={setItemsPerPage}
      />
    </div>
  )
}
```
