## Story ID
   * ID Format: US-005-002

## Story Title
   * Multi-Format Export (Excel, CSV, PDF)

## Description
  * As a Loan Processor, I want to export extracted entities in multiple formats (JSON, CSV, Excel, PDF), so that I can integrate the data into other systems and create reports

## Acceptance Criteria
  * Given user is on Results page, When they click Export button, Then dropdown shows options:
    - Download as JSON (implemented)
    - Download as CSV
    - Download as Excel
    - Generate PDF Report
  * Given user selects JSON export, When export executes, Then JSON file downloads with extraction data using Blob API
  * Given user selects CSV export, When export executes, Then CSV file downloads with flattened entity data
  * Given user selects Excel export, When export executes, Then Excel workbook downloads with:
    - Client-side generation using SheetJS (xlsx library)
    - Overview worksheet + separate worksheets per entity category
    - Formatted headers: Bold, blue background (#3B82F6), white text, frozen pane
    - Auto-sized columns to fit content
    - Number formatting: Currency as `$#,##0.00`, Percentages as `0.00%`
    - Filename pattern: `{document_name}_entities_{YYYY-MM-DD}.xlsx`
  * Given user selects PDF report, When export executes, Then formatted PDF downloads with entity data and document reference
  * Given export is in progress, When file is generating, Then:
    - Loading spinner on export button
    - Success toast: "File downloaded successfully"
    - Edge cases handled (empty categories = empty sheet with headers)

## Edge Cases
   * What happens when empty categories exist in export? Create empty sheet with headers only.
   * How does system handle export failure mid-process? Show error with partial data recovery option.
   * What happens when browser blocks download? Show instructions to allow downloads.
   * How does system handle very large datasets? Stream generation with progress indicator.
