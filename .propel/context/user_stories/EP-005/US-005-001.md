# User Story: AI Chatbot Integration

**Epic:** EP-005 - AI Chatbot & Export Functionality  
**Story ID:** US-005-001  
**Priority:** P1 - High  
**Estimate:** 8 Story Points

---

## User Story

**As a** Loan Processor  
**I want** an AI-powered chatbot on the Results page to ask questions about extracted data  
**So that** I can quickly understand and verify document information through natural language queries

---

## Acceptance Criteria

### AC1: Chatbot FAB
**Given** user is on Results page  
**When** page loads  
**Then** a Floating Action Button (FAB) appears at bottom-right to toggle chatbot

### AC2: Chatbot Overlay
**Given** user clicks the FAB  
**When** chatbot opens  
**Then** overlay window (380×520px) displays:
- Header with title ("Document Assistant" or "Dual Document Assistant")
- Scrollable message area
- Input field with multiline support (Shift+Enter for newline, Enter to send)
- Close button in header

### AC3: AI Response Integration
**Given** user sends a message  
**When** message is submitted  
**Then**:
- Typing indicator (animated dots) shows while waiting
- Response is returned from `/api/chat/` endpoint
- Markdown rendering for AI responses using ReactMarkdown
- Conversation history maintained (rolling window of last 10 messages)

### AC4: Dual-Document Context
**Given** both loan and tax documents are available  
**When** chatbot provides answers  
**Then** responses include context from both extraction results

### AC5: LLM Fallback
**Given** Azure OpenAI is primary chatbot service  
**When** primary service fails  
**Then** automatic fallback to Google Gemini (gemini-2.5-flash)
