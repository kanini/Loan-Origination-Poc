# User Story: Processing Status Page with Real-time Feedback

**Epic:** EP-005 - Processing Status & Real-time Feedback  
**Story ID:** US-005-001  
**Priority:** P0 - Critical  
**Estimate:** 5 Story Points (2 days)

---

## User Story

**As a** Financial Analyst  
**I want** a processing status page with loading indicators and automatic redirect  
**So that** I can see my document is being processed and automatically view results when complete

---

## Acceptance Criteria

### AC1: Processing Page Layout
**Given** I submit a document for extraction  
**When** I am redirected to the Processing page  
**Then**:
- Page displays centered content (max-width: 600px)
- Large animated loading spinner (Primary Blue, 64px diameter)
- Status message "Processing your document..." (H2)
- Document filename displayed
- Selected model name displayed (OpenAI GPT or Google Gemini)
- Estimated time message: "This may take 10-30 seconds depending on document complexity"
- Breadcrumb shows: "Home › Upload Document › Processing"

### AC2: Loading Animation
**Given** the Processing page is displayed  
**When** I view the loading indicator  
**Then**:
- Spinner rotates continuously at smooth 60fps
- Spinner uses Primary Blue color (#3B82F6)
- Animation is hardware-accelerated (CSS transform)
- Spinner respects prefers-reduced-motion setting
- No janky or stuttering animation

### AC3: Automatic Redirect on Success
**Given** the document processing completes successfully  
**When** the LLM returns extraction results  
**Then**:
- Spinner changes to success icon (green checkmark)
- Status message changes to "Processing complete! Redirecting to results..."
- Success icon shows for 2 seconds
- Automatic redirect to Results page (/results/{documentId})
- No user interaction required

### AC4: Error Handling
**Given** the document processing fails  
**When** the LLM API returns an error  
**Then**:
- Spinner changes to error icon (red X)
- Status message changes to "Processing failed. Please try again or select a different model."
- Error details displayed if available from API
- Three action buttons appear:
  - "Retry" (primary button) - retry with same file and model
  - "Change Model" (secondary button) - return to Upload page with file preserved
  - "Cancel" (tertiary button) - return to Dashboard
- No automatic redirect on error

### AC5: Status Polling
**Given** the Processing page is displayed  
**When** waiting for results  
**Then**:
- Status is polled from API every 2 seconds
- Polling continues until status is "Success" or "Failed"
- Polling stops after 60 seconds with timeout error
- Network errors trigger retry with exponential backoff
- User sees consistent loading state during polling

### AC6: Timeout Handling
**Given** processing takes longer than 60 seconds  
**When** the timeout threshold is reached  
**Then**:
- Error icon and message displayed
- Message: "Processing is taking longer than expected. Please try again or contact support."
- Retry and Cancel buttons available
- Processing continues in background (user can check History later)

### AC7: Page Transitions
**Given** I navigate to/from the Processing page  
**When** page transitions occur  
**Then**:
- Fade-in animation on page load (300ms)
- Smooth transition to Results page (300ms)
- No jarring or abrupt changes
- Loading state maintained during navigation

## Technical Details

### Component Implementation
```typescript
// src/pages/Processing.tsx
import { useEffect, useState } from 'react'
import { useParams, useNavigate } from 'react-router-dom'
import { useQuery } from '@tanstack/react-query'
import { Loader2, CheckCircle, XCircle } from 'lucide-react'
import { Button } from '@/components/ui/button'
import { usePageTitle } from '@/hooks/usePageTitle'
import { checkProcessingStatus } from '@/api/documents'

type ProcessingStatus = 'processing' | 'success' | 'failed' | 'timeout'

export default function Processing() {
  usePageTitle('Processing')
  const { documentId } = useParams<{ documentId: string }>()
  const navigate = useNavigate()
  const [status, setStatus] = useState<ProcessingStatus>('processing')
  const [errorMessage, setErrorMessage] = useState<string>('')

  // Poll processing status every 2 seconds
  const { data: statusData } = useQuery({
    queryKey: ['processing-status', documentId],
    queryFn: () => checkProcessingStatus(documentId!),
    refetchInterval: (data) => {
      // Stop polling if completed or failed
      if (data?.status === 'success' || data?.status === 'failed') {
        return false
      }
      return 2000 // Poll every 2 seconds
    },
    enabled: !!documentId && status === 'processing',
  })

  useEffect(() => {
    if (statusData?.status === 'success') {
      setStatus('success')
      // Redirect after 2 seconds
      setTimeout(() => {
        navigate(`/results/${documentId}`)
      }, 2000)
    } else if (statusData?.status === 'failed') {
      setStatus('failed')
      setErrorMessage(statusData.error || 'Processing failed')
    }
  }, [statusData, documentId, navigate])

  // Timeout after 60 seconds
  useEffect(() => {
    const timeout = setTimeout(() => {
      if (status === 'processing') {
        setStatus('timeout')
        setErrorMessage('Processing is taking longer than expected')
      }
    }, 60000)

    return () => clearTimeout(timeout)
  }, [status])

  const handleRetry = () => {
    setStatus('processing')
    setErrorMessage('')
    // Trigger re-processing via API
  }

  const handleChangeModel = () => {
    navigate('/upload', { state: { documentId } })
  }

  const handleCancel = () => {
    navigate('/')
  }

  return (
    <div className="min-h-[calc(100vh-200px)] flex items-center justify-center px-4">
      <div className="max-w-md w-full text-center space-y-6">
        {/* Status Icon */}
        {status === 'processing' && (
          <Loader2 className="h-16 w-16 text-primary animate-spin mx-auto" />
        )}
        {status === 'success' && (
          <CheckCircle className="h-16 w-16 text-success mx-auto" />
        )}
        {(status === 'failed' || status === 'timeout') && (
          <XCircle className="h-16 w-16 text-error mx-auto" />
        )}

        {/* Status Message */}
        <div className="space-y-2">
          <h2 className="text-h2 font-semibold text-gray-900">
            {status === 'processing' && 'Processing your document...'}
            {status === 'success' && 'Processing complete!'}
            {status === 'failed' && 'Processing failed'}
            {status === 'timeout' && 'Processing timeout'}
          </h2>

          {status === 'processing' && (
            <>
              <p className="text-base text-gray-600">
                {statusData?.filename || 'document.pdf'}
              </p>
              <p className="text-sm text-gray-500">
                Model: {statusData?.model || 'OpenAI GPT'}
              </p>
              <p className="text-sm text-gray-500">
                This may take 10-30 seconds depending on document complexity
              </p>
            </>
          )}

          {status === 'success' && (
            <p className="text-base text-gray-600">
              Redirecting to results...
            </p>
          )}

          {(status === 'failed' || status === 'timeout') && (
            <p className="text-base text-error">
              {errorMessage}
            </p>
          )}
        </div>

        {/* Action Buttons (Error State) */}
        {(status === 'failed' || status === 'timeout') && (
          <div className="flex flex-col sm:flex-row gap-3 justify-center">
            <Button onClick={handleRetry}>
              Retry
            </Button>
            <Button variant="secondary" onClick={handleChangeModel}>
              Change Model
            </Button>
            <Button variant="tertiary" onClick={handleCancel}>
              Cancel
            </Button>
          </div>
        )}
      </div>
    </div>
  )
}
```
