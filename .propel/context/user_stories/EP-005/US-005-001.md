## Story ID
   * ID Format: US-005-001

## Story Title
   * AI Chatbot Integration

## Description
  * As a Loan Processor, I want an AI-powered chatbot on the Results page to ask questions about extracted data, so that I can quickly understand and verify document information through natural language queries

## Acceptance Criteria
  * Given user is on Results page, When page loads, Then a Floating Action Button (FAB) appears at bottom-right to toggle chatbot
  * Given user clicks the FAB, When chatbot opens, Then overlay window (380×520px) displays:
    - Header with title ("Document Assistant" or "Dual Document Assistant")
    - Scrollable message area
    - Input field with multiline support (Shift+Enter for newline, Enter to send)
    - Close button in header
  * Given user sends a message, When message is submitted, Then:
    - Typing indicator (animated dots) shows while waiting
    - Response is returned from `/api/chat/` endpoint
    - Markdown rendering for AI responses using ReactMarkdown
    - Conversation history maintained (rolling window of last 10 messages)
  * Given both loan and tax documents are available, When chatbot provides answers, Then responses include context from both extraction results
  * Given Azure OpenAI is primary chatbot service, When primary service fails, Then automatic fallback to Google Gemini (gemini-2.5-flash)

## Edge Cases
   * What happens when API rate limit is exceeded? Show quota message with retry timing.
   * How does system handle very long conversation? Implement rolling window to maintain context.
   * What happens when AI response is malformed? Show error message and log for debugging.
   * How does system handle timeout on AI response? Show timeout error with retry option after 30 seconds.
