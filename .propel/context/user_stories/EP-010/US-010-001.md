# User Story: Comprehensive Error Handling System

**Epic:** EP-010 - Error Handling & User Feedback  
**Story ID:** US-010-001  
**Priority:** P0 - Critical  
**Estimate:** 5 Story Points (2 days)

---

## User Story

**As a** Financial Analyst  
**I want** clear, actionable error messages and helpful guidance when things go wrong  
**So that** I can understand what happened and know how to resolve issues without frustration

---

## Acceptance Criteria

### AC1: Inline Validation Errors
**Given** I enter invalid data in a form  
**When** validation fails  
**Then**:
- Error message appears below the input field
- Input border changes to red (#EF4444)
- Error icon displayed next to message
- Error text is specific and actionable
- Example: "Email must be a valid email address" not "Invalid input"
- Error clears when user corrects the input

### AC2: Toast Notifications
**Given** an action succeeds or fails  
**When** the action completes  
**Then**:
- Success toast: Green background, checkmark icon, white text
- Error toast: Red background, error icon, white text
- Warning toast: Amber background, warning icon, dark text
- Info toast: Blue background, info icon, white text
- Toast slides in from top-right
- Toast auto-dismisses after 5 seconds
- Manual dismiss button (X) available

### AC3: Detailed Error Messages
**Given** an error occurs  
**When** I view the error message  
**Then**:
- Message explains what went wrong in plain language
- Message suggests corrective action
- Examples:
  - "Upload failed. Please check your internet connection and try again."
  - "File size exceeds 10MB limit. Please select a smaller file."
  - "Processing failed. Try selecting a different model or contact support."
- No technical jargon or error codes shown to users
- Technical details logged for debugging

### AC4: Network Error Handling
**Given** network connectivity issues occur  
**When** API requests fail  
**Then**:
- User-friendly message: "Connection lost. Please check your internet and try again."
- Retry button available
- Automatic retry with exponential backoff (3 attempts)
- Offline indicator if persistent
- Cached data shown when available

### AC5: API Error Handling
**Given** API returns error responses  
**When** different error codes occur  
**Then**:
- 400 Bad Request: "Invalid request. Please check your input."
- 401 Unauthorized: "Session expired. Please log in again." (Phase 2)
- 403 Forbidden: "You don't have permission to perform this action."
- 404 Not Found: "Document not found. It may have been deleted."
- 500 Server Error: "Server error. Please try again later."
- 503 Service Unavailable: "Service temporarily unavailable. Please try again in a few minutes."

### AC6: File Upload Errors
**Given** file upload fails  
**When** different upload errors occur  
**Then**:
- File too large: "File size exceeds 10MB limit (File size: X MB)"
- Invalid type: "File type not supported. Please upload a PDF file"
- Network error: "Upload failed. Please check your connection and try again."
- Server error: "Upload failed. Please try again later."
- Timeout: "Upload timed out. Please try again with a smaller file."

### AC7: Processing Errors
**Given** document processing fails  
**When** LLM processing encounters errors  
**Then**:
- LLM timeout: "Processing is taking longer than expected. Please try again."
- LLM API error: "Processing failed. Try selecting a different model."
- Invalid PDF: "Unable to extract text from PDF. Please ensure the file is not corrupted."
- Rate limit: "Too many requests. Please wait a moment and try again."
- Recovery options: Retry, Change Model, Cancel

### AC8: Error Boundaries
**Given** a React component crashes  
**When** an unhandled error occurs  
**Then**:
- Error boundary catches the error
- Fallback UI displayed: "Something went wrong"
- "Refresh Page" button available
- Error details logged to console
- User can continue using other parts of the app
- Error reported to monitoring service (optional)

### AC9: Loading State Errors
**Given** data fails to load  
**When** initial page load fails  
**Then**:
- Error icon displayed
- Clear message: "Failed to load data. Please try again."
- "Refresh" or "Retry" button
- No blank screen or infinite loading
- Fallback to cached data if available

### AC10: ARIA Live Regions
**Given** errors occur  
**When** screen readers are active  
**Then**:
- Error messages announced via aria-live="assertive"
- Success messages announced via aria-live="polite"
- Screen reader users receive immediate feedback
- Announcements don't interrupt current reading
- Error focus moved to error message

## Technical Details

### Error Handling Hook
```typescript
// src/hooks/useErrorHandler.ts
import { useToast } from '@/hooks/useToast'

export function useErrorHandler() {
  const { toast } = useToast()

  const handleError = (error: any, context?: string) => {
    console.error(`Error in ${context}:`, error)

    let message = 'An unexpected error occurred. Please try again.'

    if (error.response) {
      // API error
      const status = error.response.status
      const data = error.response.data

      switch (status) {
        case 400:
          message = data.message || 'Invalid request. Please check your input.'
          break
        case 401:
          message = 'Session expired. Please log in again.'
          break
        case 403:
          message = "You don't have permission to perform this action."
          break
        case 404:
          message = 'Resource not found. It may have been deleted.'
          break
        case 500:
          message = 'Server error. Please try again later.'
          break
        case 503:
          message = 'Service temporarily unavailable. Please try again in a few minutes.'
          break
        default:
          message = data.message || 'An error occurred. Please try again.'
      }
    } else if (error.request) {
      // Network error
      message = 'Connection lost. Please check your internet and try again.'
    } else if (error.message) {
      // Client error
      message = error.message
    }

    toast({
      title: 'Error',
      description: message,
      variant: 'error',
    })

    return message
  }

  return { handleError }
}
```

### Error Boundary Component
```typescript
// src/components/common/ErrorBoundary.tsx
import { Component, ReactNode } from 'react'
import { AlertCircle } from 'lucide-react'
import { Button } from '@/components/ui/button'

interface Props {
  children: ReactNode
}

interface State {
  hasError: boolean
  error: Error | null
}

export class ErrorBoundary extends Component<Props, State> {
  constructor(props: Props) {
    super(props)
    this.state = { hasError: false, error: null }
  }

  static getDerivedStateFromError(error: Error): State {
    return { hasError: true, error }
  }

  componentDidCatch(error: Error, errorInfo: any) {
    console.error('Error boundary caught:', error, errorInfo)
    // Report to error monitoring service
  }

  handleReset = () => {
    this.setState({ hasError: false, error: null })
    window.location.reload()
  }

  render() {
    if (this.state.hasError) {
      return (
        <div className="min-h-screen flex items-center justify-center bg-gray-50 px-4">
          <div className="max-w-md w-full text-center">
            <AlertCircle className="h-16 w-16 text-error mx-auto mb-4" />
            <h1 className="text-h2 font-bold text-gray-900 mb-2">
              Something went wrong
            </h1>
            <p className="text-base text-gray-600 mb-6">
              We're sorry for the inconvenience. Please refresh the page to continue.
            </p>
            <Button onClick={this.handleReset}>
              Refresh Page
            </Button>
            {process.env.NODE_ENV === 'development' && this.state.error && (
              <details className="mt-4 text-left">
                <summary className="cursor-pointer text-sm text-gray-500">
                  Error details
                </summary>
                <pre className="mt-2 text-xs bg-gray-100 p-4 rounded overflow-auto">
                  {this.state.error.toString()}
                </pre>
              </details>
            )}
          </div>
        </div>
      )
    }

    return this.props.children
  }
}
```

### Form Validation Error Component

```typescript
// src/components/ui/form-error.tsx
import { AlertCircle } from 'lucide-react'

interface FormErrorProps {
  message?: string
}

export function FormError({ message }: FormErrorProps) {
  if (!message) return null

  return (
    <div 
      className="flex items-start gap-2 mt-1 text-sm text-error"
      role="alert"
      aria-live="assertive"
    >
      <AlertCircle className="h-4 w-4 flex-shrink-0 mt-0.5" />
      <span>{message}</span>
    </div>
  )
}
```

### API Error Interceptor
```typescript
// src/api/client.ts
import axios from 'axios'

export const apiClient = axios.create({
  baseURL: '/api',
  headers: {
    'Content-Type': 'application/json',
  },
})

// Response interceptor for error handling
apiClient.interceptors.response.use(
  (response) => response,
  (error) => {
    // Log error for debugging
    console.error('API Error:', error)

    // Enhance error with user-friendly message
    if (error.response) {
      error.userMessage = getUserFriendlyMessage(error.response.status)
    } else if (error.request) {
      error.userMessage = 'Connection lost. Please check your internet and try again.'
    } else {
      error.userMessage = 'An unexpected error occurred. Please try again.'
    }

    return Promise.reject(error)
  }
)

function getUserFriendlyMessage(status: number): string {
  const messages: Record<number, string> = {
    400: 'Invalid request. Please check your input.',
    401: 'Session expired. Please log in again.',
    403: "You don't have permission to perform this action.",
    404: 'Resource not found. It may have been deleted.',
    500: 'Server error. Please try again later.',
    503: 'Service temporarily unavailable. Please try again in a few minutes.',
  }
  return messages[status] || 'An error occurred. Please try again.'
}
```
