# User Story: Keyboard Navigation and Focus Management

**Epic:** EP-011 - Accessibility & WCAG Compliance  
**Story ID:** US-011-001  
**Priority:** P0 - Critical  
**Estimate:** 5 Story Points (2 days)

---

## User Story

**As a** user who relies on keyboard navigation  
**I want** full keyboard access to all interactive elements  
**So that** I can use the application efficiently without a mouse

---

## Acceptance Criteria

### AC1: Tab Navigation
**Given** I use the Tab key to navigate  
**When** I press Tab  
**Then**:
- Focus moves to next interactive element in logical order
- Tab order follows visual layout (left-to-right, top-to-bottom)
- All interactive elements are reachable via Tab
- Shift+Tab moves focus backward
- Focus never gets trapped (except in modals)
- Skip links allow bypassing repetitive content

### AC2: Focus Indicators
**Given** I navigate with keyboard  
**When** an element receives focus  
**Then**:
- Visible focus indicator (2px outline) appears
- Focus indicator uses Primary Blue color (#3B82F6)
- Focus indicator has sufficient contrast (≥3:1)
- Focus indicator visible on all interactive elements
- Custom focus styles match design system
- Focus indicator not removed by CSS

### AC3: Button Interactions
**Given** I focus on a button  
**When** I press Enter or Space  
**Then**:
- Button action triggers
- Enter key activates button
- Space key activates button
- Button shows active state during press
- Focus remains on button after activation (unless navigating)

### AC4: Form Interactions
**Given** I navigate through a form  
**When** I interact with form elements  
**Then**:
- Tab moves between form fields in logical order
- Enter submits form (when appropriate)
- Arrow keys navigate radio groups and select options
- Space toggles checkboxes
- Esc clears autocomplete suggestions
- Form validation errors announced

### AC5: Modal Focus Trap
**Given** a modal is open  
**When** I press Tab  
**Then**:
- Focus stays within modal (focus trap)
- Tab cycles through modal elements only
- Shift+Tab cycles backward within modal
- Esc key closes modal
- Focus returns to trigger element on close
- Background content not focusable

### AC6: Dropdown/Menu Navigation
**Given** I open a dropdown menu  
**When** I use keyboard  
**Then**:
- Enter or Space opens dropdown
- Arrow Down moves to next item
- Arrow Up moves to previous item
- Home moves to first item
- End moves to last item
- Enter selects current item
- Esc closes dropdown
- Type-ahead search works (optional)

### AC7: Skip Navigation Links
**Given** I am a keyboard user  
**When** I load a page  
**Then**:
- "Skip to main content" link is first focusable element
- Skip link visible on focus
- Skip link jumps to main content area
- Skip link bypasses repetitive navigation
- Multiple skip links for complex layouts (optional)

### AC8: Table Navigation
**Given** I navigate a data table  
**When** I use keyboard  
**Then**:
- Tab moves through interactive cells
- Arrow keys navigate cells (optional enhancement)
- Enter activates inline editing
- Esc cancels editing
- Sortable headers activated with Enter/Space

### AC9: Keyboard Shortcuts
**Given** I want to use shortcuts  
**When** I press key combinations  
**Then**:
- Ctrl/Cmd + S saves (if applicable)
- Ctrl/Cmd + K opens search (if applicable)
- Esc closes modals, dropdowns, overlays
- Arrow keys navigate carousels, PDFs
- Shortcuts don't conflict with browser shortcuts
- Shortcuts documented in help section

### AC10: Focus Management on Route Change
**Given** I navigate to a new page  
**When** the route changes  
**Then**:
- Focus moves to page heading or main content
- Focus not lost during navigation
- Announcement made to screen readers
- Scroll position reset to top
- Previous focus restored on back navigation

### Focus Trap Hook
```typescript
// src/hooks/useFocusTrap.ts
import { useEffect, useRef } from 'react'

export function useFocusTrap(isActive: boolean) {
  const containerRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    if (!isActive) return

    const container = containerRef.current
    if (!container) return

    const focusableElements = container.querySelectorAll(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    )

    const firstElement = focusableElements[0] as HTMLElement
    const lastElement = focusableElements[focusableElements.length - 1] as HTMLElement

    const handleTabKey = (e: KeyboardEvent) => {
      if (e.key !== 'Tab') return

      if (e.shiftKey) {
        if (document.activeElement === firstElement) {
          e.preventDefault()
          lastElement?.focus()
        }
      } else {
        if (document.activeElement === lastElement) {
          e.preventDefault()
          firstElement?.focus()
        }
      }
    }

    container.addEventListener('keydown', handleTabKey)
    firstElement?.focus()

    return () => {
      container.removeEventListener('keydown', handleTabKey)
    }
  }, [isActive])

  return containerRef
}

// Usage in Modal
const containerRef = useFocusTrap(isOpen)

<div ref={containerRef} role="dialog">
  {/* Modal content */}
</div>
```

### Focus Management on Route Change
```typescript
// src/components/layout/PageLayout.tsx
import { useEffect, useRef } from 'react'
import { useLocation } from 'react-router-dom'

export function PageLayout({ children }: { children: React.ReactNode }) {
  const location = useLocation()
  const mainRef = useRef<HTMLElement>(null)

  useEffect(() => {
    // Focus main content on route change
    mainRef.current?.focus()
    // Scroll to top
    window.scrollTo(0, 0)
  }, [location.pathname])

  return (
    <main
      ref={mainRef}
      id="main-content"
      tabIndex={-1}
      className="outline-none"
    >
      {children}
    </main>
  )
}
```

### Keyboard Event Handler Hook
```typescript
// src/hooks/useKeyboardShortcuts.ts
import { useEffect } from 'react'

interface Shortcut {
  key: string
  ctrlKey?: boolean
  shiftKey?: boolean
  callback: () => void
}

export function useKeyboardShortcuts(shortcuts: Shortcut[]) {
  useEffect(() => {
    const handleKeyDown = (e: KeyboardEvent) => {
      shortcuts.forEach(({ key, ctrlKey, shiftKey, callback }) => {
        const ctrlMatch = ctrlKey === undefined || e.ctrlKey === ctrlKey || e.metaKey === ctrlKey
        const shiftMatch = shiftKey === undefined || e.shiftKey === shiftKey

        if (e.key === key && ctrlMatch && shiftMatch) {
          e.preventDefault()
          callback()
        }
      })
    }

    window.addEventListener('keydown', handleKeyDown)
    return () => window.removeEventListener('keydown', handleKeyDown)
  }, [shortcuts])
}

// Usage
useKeyboardShortcuts([
  { key: 'Escape', callback: () => setModalOpen(false) },
  { key: 's', ctrlKey: true, callback: handleSave },
])
```
