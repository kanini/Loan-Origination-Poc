# User Story: Results Display with Card/Table Views

**Epic:** EP-004 - Results Display & Entity Management  
**Story ID:** US-004-001  
**Priority:** P0 - Critical  
**Estimate:** 13 Story Points

---

## User Story

**As a** Loan Processor  
**I want** to view extracted entities in both card and table formats with dual-document tab navigation  
**So that** I can efficiently review and verify extracted data from loan applications and tax returns

---

## Acceptance Criteria

### AC1: Document Information Header
**Given** user navigates to Results page  
**When** the page loads  
**Then** header displays:
- Filename, upload date, model used, processing time
- Action buttons: Edit, Export, Delete

### AC2: Dual-Document Tab Navigation
**Given** both loan and tax documents were processed  
**When** Results page loads  
**Then** tab bar allows switching between:
- "Loan Application" tab
- "Tax Return Form" tab

### AC3: Card View (Default)
**Given** Card View is selected  
**When** entities display  
**Then** each category shows:
- Separate card for each entity category (Lender, Borrower, Loan Terms, etc.)
- Field label + value pairs
- Nested sub-sections with numbered labels (e.g., "Taxpayer 1", "Taxpayer 2")
- Confidence scores when available
- Click-to-edit inline editing

### AC4: Table View
**Given** Table View is selected  
**When** entities display  
**Then** each category shows as table with:
- Columns: Field Name, Extracted Value
- Sortable columns
- Quick-search filter
- Sticky headers, row striping

### AC5: View Toggle
**Given** user wants to switch views  
**When** toggle is clicked  
**Then** view switches between Card and Table with state persisted in localStorage

### AC6: Empty Field Handling
**Given** some fields have no extracted value  
**When** displaying results  
**Then** placeholder "--" shown with muted styling
