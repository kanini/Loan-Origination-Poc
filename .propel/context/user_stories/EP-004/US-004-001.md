## Story ID
   * ID Format: US-004-001

## Story Title
   * Results Display with Card/Table Views

## Description
  * As a Loan Processor, I want to view extracted entities in both card and table formats with dual-document tab navigation, so that I can efficiently review and verify extracted data from loan applications and tax returns

## Acceptance Criteria
  * Given user navigates to Results page, When the page loads, Then header displays:
    - Filename, upload date, model used, processing time
    - Action buttons: Edit, Export, Delete
  * Given both loan and tax documents were processed, When Results page loads, Then tab bar allows switching between:
    - "Loan Application" tab
    - "Tax Return Form" tab
  * Given Card View is selected, When entities display, Then each category shows:
    - Separate card for each entity category (Lender, Borrower, Loan Terms, etc.)
    - Field label + value pairs
    - Nested sub-sections with numbered labels (e.g., "Taxpayer 1", "Taxpayer 2")
    - Confidence scores when available
    - Click-to-edit inline editing
  * Given Table View is selected, When entities display, Then each category shows as table with:
    - Columns: Field Name, Extracted Value
    - Sortable columns
    - Quick-search filter
    - Sticky headers, row striping
  * Given user wants to switch views, When toggle is clicked, Then view switches between Card and Table with state persisted in localStorage
  * Given some fields have no extracted value, When displaying results, Then placeholder "--" shown with muted styling

## Edge Cases
   * What happens when no entities are extracted? Show empty state with troubleshooting guidance.
   * How does system handle confidence score below threshold? Highlight field with warning indicator.
   * What happens when inline edit fails to save? Show error toast and revert field value.
   * How does system handle very long field values? Truncate with expand option.
