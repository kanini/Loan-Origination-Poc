## Story ID
   * ID Format: US-004-002

## Story Title
   * PDF Viewer & Split Panel Layout

## Description
  * As a Loan Processor, I want to view the original PDF alongside extracted entities in a split-panel layout, so that I can verify extracted data against the source document

## Acceptance Criteria
  * Given Results page loads, When PDF viewer panel renders, Then it displays:
    - Original PDF document using react-pdf
    - Zoom controls: Zoom In, Zoom Out, Fit to Width, Fit to Page
    - Page navigation: Previous, Next, "Page X of Y"
    - Full-screen option
  * Given user is on desktop (≥1024px), When Results page renders, Then layout shows:
    - 60% width for entity display (left)
    - 40% width for PDF viewer (right)
  * Given user is on tablet or mobile (<1024px), When Results page renders, Then tabbed interface allows switching between:
    - Entities tab
    - PDF Viewer tab
  * Given table view is active on mobile, When tables render, Then horizontal scrolling enabled with full-width on desktop

## Edge Cases
   * What happens when PDF fails to load? Show placeholder with error message and retry option.
   * How does system handle PDF with hundreds of pages? Virtualize page rendering for performance.
   * What happens when PDF is password-protected? Show prompt for password or error message.
   * How does system handle corrupted PDF? Show error with option to re-upload.
