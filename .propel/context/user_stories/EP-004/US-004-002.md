# User Story: PDF Viewer & Split Panel Layout

**Epic:** EP-004 - Results Display & Entity Management  
**Story ID:** US-004-002  
**Priority:** P1 - High  
**Estimate:** 8 Story Points

---

## User Story

**As a** Loan Processor  
**I want** to view the original PDF alongside extracted entities in a split-panel layout  
**So that** I can verify extracted data against the source document

---

## Acceptance Criteria

### AC1: PDF Viewer Integration
**Given** Results page loads  
**When** PDF viewer panel renders  
**Then** it displays:
- Original PDF document using react-pdf
- Zoom controls: Zoom In, Zoom Out, Fit to Width, Fit to Page
- Page navigation: Previous, Next, "Page X of Y"
- Full-screen option

### AC2: Split Panel Layout (Desktop)
**Given** user is on desktop (≥1024px)  
**When** Results page renders  
**Then** layout shows:
- 60% width for entity display (left)
- 40% width for PDF viewer (right)

### AC3: Tabbed Layout (Tablet/Mobile)
**Given** user is on tablet or mobile (<1024px)  
**When** Results page renders  
**Then** tabbed interface allows switching between:
- Entities tab
- PDF Viewer tab

### AC4: Responsive Tables
**Given** table view is active on mobile  
**When** tables render  
**Then** horizontal scrolling enabled with full-width on desktop
