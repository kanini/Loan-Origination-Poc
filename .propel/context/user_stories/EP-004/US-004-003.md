# User Story: Upload Progress and Form Submission

**Epic:** EP-004 - Document Upload Workflow  
**Story ID:** US-004-003  
**Priority:** P0 - Critical  
**Estimate:** 5 Story Points (2 days)

---

## User Story

**As a** Financial Analyst  
**I want** to see upload progress and submit my document for processing  
**So that** I know the upload is progressing and can track when processing begins

---

## Acceptance Criteria

### AC1: Form Validation Before Submission
**Given** I am on the Upload page  
**When** I click "Extract Entities" button  
**Then**:
- System validates that a file is selected
- System validates that a model is selected
- If file missing: Error message "Please select a PDF file"
- If model missing: Error message "Please choose an LLM model"
- If both missing: Both error messages display
- Button remains disabled until all validations pass

### AC2: Upload Progress Bar
**Given** I submit a valid file and model selection  
**When** the upload begins  
**Then**:
- Progress bar appears below submit button
- Progress bar shows percentage (0-100%)
- Progress updates in real-time during upload
- File size and uploaded amount displayed (e.g., "2.5 MB / 5.0 MB")
- Progress bar uses Primary Blue color
- Animation is smooth (60fps)

### AC3: Upload Button States
**Given** I interact with the upload process  
**When** different states occur  
**Then**:
- Default state: "Extract Entities" button enabled when file and model selected
- Uploading state: Button shows "Uploading..." with spinner, disabled
- Success state: Button shows "Processing..." briefly before redirect
- Error state: Button returns to "Extract Entities", enabled for retry

### AC4: Cancel Upload Option
**Given** upload is in progress  
**When** I click "Cancel" button  
**Then**:
- Upload is aborted immediately
- Progress bar disappears
- File selection and model selection preserved
- User can modify selections or retry upload
- No partial upload sent to server

### AC5: Upload Success Handling
**Given** file upload completes successfully  
**When** the server confirms receipt  
**Then**:
- Success toast notification: "Document uploaded successfully. Processing..."
- Progress bar shows 100% briefly
- Automatic redirect to Processing page (/processing/{documentId})
- Document ID passed to Processing page
- Redirect occurs within 1 second of completion

### AC6: Upload Error Handling
**Given** upload fails  
**When** an error occurs  
**Then**:
- Error toast notification with specific message
- Network error: "Upload failed. Please check your connection and try again."
- Server error: "Upload failed. Please try again later."
- File too large: "File size exceeds server limit."
- Progress bar shows error state (red color)
- Retry button available

### AC7: Step Progress Indicator
**Given** I am in the upload workflow  
**When** I view the page  
**Then**:
- Step indicator shows: Step 1 (Select File) → Step 2 (Choose Model) → Step 3 (Confirm)
- Current step highlighted in Primary Blue
- Completed steps show checkmark icon
- Future steps show gray color
- Step indicator updates as user progresses

### AC8: Form Data Submission
**Given** upload completes  
**When** form is submitted to backend  
**Then**:
- File uploaded as multipart/form-data
- Model selection sent as form field
- Document metadata sent (filename, size, upload timestamp)
- CSRF token included in request headers
- Request sent to `/api/documents/upload/` endpoint

### AC9: Network Resilience
**Given** network issues occur during upload  
**When** connection is interrupted  
**Then**:
- Upload pauses automatically
- User notified: "Connection lost. Upload paused."
- Upload resumes when connection restored
- Progress preserved (resume from last byte)
- Maximum 3 retry attempts with exponential backoff

### AC10: Large File Handling
**Given** I upload a large file (5-10MB)  
**When** upload is in progress  
**Then**:
- Upload doesn't block UI (runs asynchronously)
- Progress updates smoothly without freezing
- Memory usage remains reasonable
- Upload completes within 30 seconds on 3G connection
- Chunked upload for files >5MB (optional enhancement)

## Technical Details

### Upload Implementation
```typescript
// src/pages/Upload.tsx
import { useState } from 'react'
import { useNavigate } from 'react-router-dom'
import { useMutation } from '@tanstack/react-query'
import { FileUploadZone } from '@/components/upload/FileUploadZone'
import { ModelSelection } from '@/components/upload/ModelSelection'
import { ProgressBar } from '@/components/ui/progress-bar'
import { Button } from '@/components/ui/button'
import { useToast } from '@/hooks/useToast'
import { uploadDocument } from '@/api/documents'

export default function Upload() {
  const navigate = useNavigate()
  const { toast } = useToast()
  const [selectedFile, setSelectedFile] = useState<File | null>(null)
  const [selectedModel, setSelectedModel] = useState<'openai' | 'gemini' | null>(null)
  const [uploadProgress, setUploadProgress] = useState(0)
  const [fileError, setFileError] = useState('')
  const [modelError, setModelError] = useState('')

  const uploadMutation = useMutation({
    mutationFn: (data: FormData) => uploadDocument(data, {
      onUploadProgress: (progressEvent) => {
        const percentCompleted = Math.round(
          (progressEvent.loaded * 100) / (progressEvent.total || 1)
        )
        setUploadProgress(percentCompleted)
      },
    }),
    onSuccess: (data) => {
      toast({
        title: 'Success',
        description: 'Document uploaded successfully. Processing...',
        variant: 'success',
      })
      setTimeout(() => {
        navigate(`/processing/${data.documentId}`)
      }, 1000)
    },
    onError: (error: any) => {
      const message = error.response?.data?.message || 'Upload failed. Please try again.'
      toast({
        title: 'Upload Failed',
        description: message,
        variant: 'error',
      })
      setUploadProgress(0)
    },
  })

  const handleSubmit = () => {
    // Validation
    let hasError = false
    
    if (!selectedFile) {
      setFileError('Please select a PDF file')
      hasError = true
    } else {
      setFileError('')
    }

    if (!selectedModel) {
      setModelError('Please choose an LLM model')
      hasError = true
    } else {
      setModelError('')
    }

    if (hasError) return

    // Create FormData
    const formData = new FormData()
    formData.append('file', selectedFile!)
    formData.append('model', selectedModel!)
    formData.append('filename', selectedFile!.name)
    formData.append('fileSize', selectedFile!.size.toString())

    // Submit
    uploadMutation.mutate(formData)
  }

  const handleCancel = () => {
    if (uploadMutation.isPending) {
      uploadMutation.reset()
      setUploadProgress(0)
    } else {
      navigate('/')
    }
  }

  return (
    <div className="max-w-3xl mx-auto px-4 py-8">
      <h1 className="text-h1 font-bold mb-8">Upload Document</h1>

      {/* Step Indicator */}
      <StepIndicator currentStep={selectedFile ? (selectedModel ? 3 : 2) : 1} />

      {/* File Upload */}
      <div className="mb-8">
        <FileUploadZone
          selectedFile={selectedFile}
          onFileSelect={setSelectedFile}
          error={fileError}
        />
      </div>

      {/* Model Selection */}
      <div className="mb-8">
        <ModelSelection
          selectedModel={selectedModel}
          onModelSelect={setSelectedModel}
          error={modelError}
        />
      </div>

      {/* Upload Progress */}
      {uploadMutation.isPending && (
        <div className="mb-8">
          <ProgressBar
            value={uploadProgress}
            max={100}
            label={`Uploading: ${uploadProgress}%`}
          />
          <p className="text-sm text-gray-600 mt-2">
            {formatBytes(selectedFile!.size * uploadProgress / 100)} / {formatBytes(selectedFile!.size)}
          </p>
        </div>
      )}

      {/* Action Buttons */}
      <div className="flex gap-4">
        <Button
          variant="secondary"
          onClick={handleCancel}
          disabled={uploadMutation.isPending}
        >
          Cancel
        </Button>
        <Button
          onClick={handleSubmit}
          disabled={!selectedFile || !selectedModel || uploadMutation.isPending}
          loading={uploadMutation.isPending}
        >
          {uploadMutation.isPending ? 'Uploading...' : 'Extract Entities'}
        </Button>
      </div>
    </div>
  )
}

function formatBytes(bytes: number): string {
  if (bytes < 1024) return `${bytes} B`
  if (bytes < 1024 * 1024) return `${(bytes / 1024).toFixed(1)} KB`
  return `${(bytes / (1024 * 1024)).toFixed(1)} MB`
}
```

### API Upload Function
```typescript
// src/api/documents.ts
import axios from 'axios'

export async function uploadDocument(
  formData: FormData,
  options?: { onUploadProgress?: (progressEvent: any) => void }
) {
  const response = await axios.post('/api/documents/upload/', formData, {
    headers: {
      'Content-Type': 'multipart/form-data',
    },
    onUploadProgress: options?.onUploadProgress,
  })
  return response.data
}
```
