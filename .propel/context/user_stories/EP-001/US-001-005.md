# User Story: Code Splitting and Lazy Loading Infrastructure

**Epic:** EP-001 - Frontend Foundation & Design System  
**Story ID:** US-001-005  
**Priority:** P0 - Critical  
**Estimate:** 5 Story Points (2 days)

---

## User Story

**As a** Frontend Developer  
**I want** to implement code splitting with React.lazy() and Suspense boundaries  
**So that** the initial JavaScript bundle size stays under 300KB gzipped and pages load faster with optimized chunks

---

## Acceptance Criteria

### AC1: Route-Based Code Splitting
**Given** the application has multiple pages  
**When** I implement route-based code splitting  
**Then**:
- Dashboard, Upload, Processing, Results, and History pages are lazy-loaded
- Each page is in a separate JavaScript chunk
- Initial bundle only includes core dependencies (React, Router, common components)
- Route chunks load on-demand when user navigates to that page
- Navigation shows loading state during chunk loading

### AC2: Suspense Boundaries
**Given** lazy-loaded components are used  
**When** I add Suspense boundaries  
**Then**:
- Each lazy-loaded route has a Suspense wrapper
- Fallback component shows skeleton screen matching page layout
- Suspense boundaries prevent blank screens during loading
- Nested Suspense boundaries for granular loading states
- Error boundaries catch chunk loading failures

### AC3: Dynamic Imports for Heavy Components
**Given** the application uses heavy third-party libraries  
**When** I implement dynamic imports  
**Then**:
- PDF viewer (react-pdf) is lazy-loaded only on Results page
- TanStack Table is lazy-loaded only when table view is active
- Excel export library (SheetJS) is lazy-loaded only when export is triggered
- Framer Motion is code-split for animations
- Each heavy component is in a separate chunk

### AC4: Bundle Size Optimization
**Given** the application is built for production  
**When** I analyze the bundle size  
**Then**:
- Initial JavaScript bundle is <300KB gzipped
- Vendor chunk (React, React DOM, Router) is separate and cacheable
- Common components chunk is shared across routes
- Each route chunk is <100KB gzipped
- Total bundle size is optimized with tree-shaking and minification

### AC5: Preloading Strategy
**Given** users navigate between pages  
**When** I implement preloading  
**Then**:
- Next likely route chunks are preloaded on hover/focus
- Critical routes (Dashboard, Upload) are prefetched after initial load
- Preloading uses `<link rel="prefetch">` for low-priority loading
- Preloading respects user's network conditions (no preload on slow 3G)

### AC6: Loading States
**Given** chunks are loading  
**When** users navigate or trigger lazy components  
**Then**:
- Skeleton screens match final page layout
- Loading spinners show for component-level lazy loading
- Progress indicators show for large chunk downloads
- Timeout handling for slow connections (show error after 10s)
- Retry mechanism for failed chunk loads

## Technical Details

### Route-Based Code Splitting
```typescript
// src/App.tsx
import { lazy, Suspense } from 'react'
import { BrowserRouter, Routes, Route } from 'react-router-dom'
import { PageSkeleton } from '@/components/common/PageSkeleton'
import { ErrorBoundary } from '@/components/common/ErrorBoundary'

// Lazy-loaded route components
const Dashboard = lazy(() => import('@/pages/Dashboard'))
const Upload = lazy(() => import('@/pages/Upload'))
const Processing = lazy(() => import('@/pages/Processing'))
const Results = lazy(() => import('@/pages/Results'))
const History = lazy(() => import('@/pages/History'))

function App() {
  return (
    <BrowserRouter>
      <ErrorBoundary>
        <Routes>
          <Route
            path="/"
            element={
              <Suspense fallback={<PageSkeleton />}>
                <Dashboard />
              </Suspense>
            }
          />
          <Route
            path="/upload"
            element={
              <Suspense fallback={<PageSkeleton />}>
                <Upload />
              </Suspense>
            }
          />
          {/* Other routes... */}
        </Routes>
      </ErrorBoundary>
    </BrowserRouter>
  )
}
```

### Dynamic Import for Heavy Components
```typescript
// src/pages/Results.tsx
import { lazy, Suspense, useState } from 'react'
import { Skeleton } from '@/components/ui/skeleton'

// Lazy-load PDF viewer only when needed
const PDFViewer = lazy(() => import('@/components/PDFViewer'))

// Lazy-load TanStack Table only in table view
const DataTable = lazy(() => import('@/components/DataTable'))

export default function Results() {
  const [viewMode, setViewMode] = useState<'card' | 'table'>('card')

  return (
    <div>
      {/* Card view - no heavy dependencies */}
      {viewMode === 'card' && <CardView />}
      
      {/* Table view - lazy-load TanStack Table */}
      {viewMode === 'table' && (
        <Suspense fallback={<Skeleton className="h-96" />}>
          <DataTable />
        </Suspense>
      )}
      
      {/* PDF viewer - lazy-loaded */}
      <Suspense fallback={<Skeleton className="h-screen" />}>
        <PDFViewer />
      </Suspense>
    </div>
  )
}
```

### Vite Bundle Configuration
```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          // Vendor chunk for React and core libraries
          vendor: ['react', 'react-dom', 'react-router-dom'],
          // UI components chunk
          ui: ['@radix-ui/react-dialog', '@radix-ui/react-dropdown-menu'],
          // Heavy libraries in separate chunks
          pdf: ['react-pdf', 'pdfjs-dist'],
          table: ['@tanstack/react-table', '@tanstack/react-virtual'],
          excel: ['xlsx'],
        },
      },
    },
    chunkSizeWarningLimit: 500, // Warn if chunk exceeds 500KB
  },
})
```

### Preloading Strategy
```typescript
// src/utils/preload.ts
export function preloadRoute(routePath: string) {
  const routeMap: Record<string, () => Promise<any>> = {
    '/dashboard': () => import('@/pages/Dashboard'),
    '/upload': () => import('@/pages/Upload'),
    '/results': () => import('@/pages/Results'),
    '/history': () => import('@/pages/History'),
  }

  const preloadFn = routeMap[routePath]
  if (preloadFn) {
    preloadFn()
  }
}

// Usage in navigation component
<Link 
  to="/upload" 
  onMouseEnter={() => preloadRoute('/upload')}
  onFocus={() => preloadRoute('/upload')}
>
  Upload
</Link>
```
