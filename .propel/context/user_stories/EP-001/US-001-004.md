# User Story: Reusable Component Library Implementation

**Epic:** EP-001 - Frontend Foundation & Design System  
**Story ID:** US-001-004  
**Priority:** P0 - Critical  
**Estimate:** 8 Story Points (3-4 days)

---

## User Story

**As a** Frontend Developer  
**I want** to build a reusable component library with buttons, form inputs, cards, tables, modals, toasts, loading indicators, and navigation components  
**So that** I can maintain consistency across the application and accelerate development with pre-built, tested components

---

## Acceptance Criteria

### AC1: Button Components
**Given** I need button components  
**When** I create button variants  
**Then**:
- Primary button: Blue background (#3B82F6), white text, hover scale 1.02
- Secondary button: White background, blue border, blue text
- Tertiary button: Transparent background, blue text, no border
- Danger button: Red background (#EF4444), white text
- All buttons support sizes: Small (32px), Medium (40px), Large (48px)
- All buttons support states: Default, Hover, Focus, Active, Disabled, Loading
- Loading state shows spinner inside button with disabled state

### AC2: Form Input Components
**Given** I need form input components  
**When** I create input variants  
**Then**:
- Text Input: Border, padding, focus state with blue outline
- File Input: Custom styled with upload icon
- Select/Dropdown: Custom styled with chevron icon
- Checkbox: Custom styled with checkmark icon
- Radio Button: Custom styled with dot indicator
- Toggle Switch: Sliding toggle with on/off states
- All inputs support validation states: Default, Error (red border + message), Success (green border)
- All inputs have associated labels with proper `for` attribute

### AC3: Card Components
**Given** I need card components  
**When** I create card variants  
**Then**:
- Basic Card: White background, shadow-sm, rounded corners, padding
- Card with Header: Includes header section with title and optional actions
- Card with Footer: Includes footer section for actions
- Stat Card: Displays metric with label, value, and optional trend indicator
- Hover effect: Shadow lift on hover (shadow-md)
- All cards use consistent spacing (padding: 24px)

### AC4: Table Components
**Given** I need table components  
**When** I create table structure  
**Then**:
- Table wrapper with horizontal scroll on mobile
- Table header with sticky positioning
- Table rows with alternating background (row striping)
- Table cells with proper padding and alignment
- Sortable column headers with sort indicator icons
- Empty state for tables with no data
- Loading state with skeleton rows

### AC5: Modal/Dialog Components
**Given** I need modal components  
**When** I create modal variants  
**Then**:
- Modal overlay: Semi-transparent black background (rgba(0,0,0,0.5))
- Modal content: White background, shadow-xl, centered, max-width 600px
- Modal header: Title and close button (X icon)
- Modal body: Scrollable content area
- Modal footer: Action buttons (Cancel, Confirm)
- Close on Esc key press
- Close on overlay click
- Focus trap within modal when open
- Confirmation dialog variant for destructive actions

### AC6: Toast Notification Components
**Given** I need toast notifications  
**When** I create toast variants  
**Then**:
- Success toast: Green background, checkmark icon, white text
- Error toast: Red background, error icon, white text
- Warning toast: Amber background, warning icon, dark text
- Info toast: Blue background, info icon, white text
- Slide in from top-right corner
- Auto-dismiss after 5 seconds
- Manual dismiss with close button
- Stack multiple toasts vertically
- Accessible with ARIA live regions

### AC7: Loading Indicator Components
**Given** I need loading indicators  
**When** I create loading variants  
**Then**:
- Spinner: Circular spinner with primary blue color, sizes: Small, Medium, Large
- Skeleton Screen: Gray placeholder matching final content layout
- Progress Bar: Horizontal bar with percentage indicator (0-100%)
- All loading indicators are accessible with ARIA labels

### AC8: Navigation Components
**Given** I need navigation components  
**When** I create navigation variants  
**Then**:
- Top Navigation Bar: Fixed header with logo, menu items, user profile icon
- Breadcrumb: Shows page hierarchy with separator icons
- Hamburger Menu: Mobile navigation toggle (visible <768px)
- Active state indicator: Highlighted menu item for current page
- All navigation is keyboard accessible

### Example Button Component
```typescript
// src/components/ui/button.tsx
import { ButtonHTMLAttributes, forwardRef } from 'react'
import { cva, type VariantProps } from 'class-variance-authority'
import { cn } from '@/lib/utils'

const buttonVariants = cva(
  "inline-flex items-center justify-center rounded-md font-medium transition-all focus:outline-none focus:ring-2 focus:ring-offset-2 disabled:opacity-50 disabled:pointer-events-none",
  {
    variants: {
      variant: {
        primary: "bg-primary text-white hover:scale-102 focus:ring-primary",
        secondary: "bg-white text-primary border border-primary hover:bg-gray-50",
        tertiary: "text-primary hover:bg-gray-50",
        danger: "bg-error text-white hover:scale-102 focus:ring-error",
      },
      size: {
        sm: "h-8 px-3 text-sm",
        md: "h-10 px-4 text-base",
        lg: "h-12 px-6 text-lg",
      },
    },
    defaultVariants: {
      variant: "primary",
      size: "md",
    },
  }
)

interface ButtonProps extends ButtonHTMLAttributes<HTMLButtonElement>, VariantProps<typeof buttonVariants> {
  loading?: boolean
}

const Button = forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant, size, loading, children, ...props }, ref) => {
    return (
      <button
        className={cn(buttonVariants({ variant, size, className }))}
        ref={ref}
        disabled={loading || props.disabled}
        {...props}
      >
        {loading && <Spinner className="mr-2 h-4 w-4" />}
        {children}
      </button>
    )
  }
)

export { Button, buttonVariants }
```
