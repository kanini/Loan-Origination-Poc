# User Story: Framer Motion Animations and Transitions

**Epic:** EP-009 - Interaction Design & Micro-interactions  
**Story ID:** US-009-001  
**Priority:** P1 - High  
**Estimate:** 5 Story Points (2 days)

---

## User Story

**As a** Financial Analyst  
**I want** smooth animations and transitions throughout the application  
**So that** the interface feels polished, responsive, and provides visual feedback for my actions

---

## Acceptance Criteria

### AC1: Framer Motion Installation
**Given** the React project is set up  
**When** I install Framer Motion  
**Then**:
- framer-motion v11.0+ installed as dependency
- No version conflicts with React 18+
- TypeScript types available
- Library size impact acceptable (~60KB gzipped)

### AC2: Page Transitions
**Given** I navigate between pages  
**When** routes change  
**Then**:
- Fade-in animation on page load (300ms duration)
- Smooth opacity transition from 0 to 1
- No jarring or abrupt page changes
- Animation uses hardware acceleration (GPU)
- Transition completes within 300ms (NFR-013)

### AC3: Button Hover Effects
**Given** I hover over buttons  
**When** the mouse enters button area  
**Then**:
- Button scales to 1.02 (2% larger)
- Transition duration: 150ms
- Smooth easing function (ease-in-out)
- Scale returns to 1.0 on mouse leave
- Effect applies to all button variants

### AC4: Card Hover Effects
**Given** I hover over cards  
**When** the mouse enters card area  
**Then**:
- Card shadow lifts from shadow-sm to shadow-md
- Transition duration: 200ms
- Smooth easing function
- Shadow returns to shadow-sm on mouse leave
- Effect applies to all card components

### AC5: Modal/Dialog Animations
**Given** I open or close a modal  
**When** the modal state changes  
**Then**:
- Modal fades in with scale animation (0.95 to 1.0)
- Overlay fades in from opacity 0 to 0.5
- Animation duration: 200ms
- Modal fades out and scales down on close
- Smooth entrance and exit animations

### AC6: Toast Notification Animations
**Given** a toast notification appears  
**When** the toast is triggered  
**Then**:
- Toast slides in from top-right corner
- Slide distance: 100px
- Animation duration: 300ms
- Toast auto-dismisses after 5 seconds
- Exit animation: fade out and slide up
- Multiple toasts stack with stagger animation

### AC7: Form Validation Shake
**Given** form validation fails  
**When** an error occurs  
**Then**:
- Input field shakes horizontally
- Shake distance: ±10px
- Animation duration: 400ms
- Shake repeats 3 times
- Red border appears simultaneously
- Error message fades in below field

### AC8: Loading State Animations
**Given** content is loading  
**When** async operations occur  
**Then**:
- Spinner rotates continuously at 360°
- Rotation duration: 1 second (infinite loop)
- Skeleton screens pulse with opacity animation
- Pulse duration: 2 seconds (infinite loop)
- Progress bars animate smoothly from 0 to 100%

### AC9: Accessibility - Reduced Motion
**Given** user has prefers-reduced-motion enabled  
**When** animations are triggered  
**Then**:
- All animations respect prefers-reduced-motion setting
- Animations reduced to instant transitions
- No motion sickness triggers
- Functionality remains intact
- Accessibility compliance maintained

### AC10: Performance Optimization
**Given** animations are running  
**When** I monitor performance  
**Then**:
- Animations run at 60fps
- No janky or stuttering motion
- Hardware acceleration used (transform, opacity)
- No layout thrashing
- Smooth on low-end devices

## Technical Details

### Installation
```bash
npm install framer-motion@^11.0.0
```

### Page Transition Component
```typescript
// src/components/common/PageTransition.tsx
import { motion } from 'framer-motion'

const pageVariants = {
  initial: {
    opacity: 0,
  },
  animate: {
    opacity: 1,
    transition: {
      duration: 0.3,
      ease: 'easeInOut',
    },
  },
  exit: {
    opacity: 0,
    transition: {
      duration: 0.2,
    },
  },
}

export function PageTransition({ children }: { children: React.ReactNode }) {
  return (
    <motion.div
      initial="initial"
      animate="animate"
      exit="exit"
      variants={pageVariants}
    >
      {children}
    </motion.div>
  )
}
```

### Modal Animation
```typescript
// src/components/ui/dialog.tsx (enhanced)
import { motion, AnimatePresence } from 'framer-motion'

const overlayVariants = {
  hidden: { opacity: 0 },
  visible: { opacity: 0.5 },
}

const modalVariants = {
  hidden: { opacity: 0, scale: 0.95 },
  visible: { 
    opacity: 1, 
    scale: 1,
    transition: { duration: 0.2 },
  },
  exit: {
    opacity: 0,
    scale: 0.95,
    transition: { duration: 0.15 },
  },
}

export function Dialog({ open, children }: DialogProps) {
  return (
    <AnimatePresence>
      {open && (
        <>
          <motion.div
            variants={overlayVariants}
            initial="hidden"
            animate="visible"
            exit="hidden"
            className="fixed inset-0 bg-black"
          />
          <motion.div
            variants={modalVariants}
            initial="hidden"
            animate="visible"
            exit="exit"
            className="fixed inset-0 flex items-center justify-center"
          >
            <div className="bg-white rounded-lg p-6 max-w-md">
              {children}
            </div>
          </motion.div>
        </>
      )}
    </AnimatePresence>
  )
}
```

### Toast Animation
```typescript
// src/components/ui/toast.tsx (enhanced)
import { motion } from 'framer-motion'

const toastVariants = {
  initial: { 
    opacity: 0, 
    x: 100,
  },
  animate: { 
    opacity: 1, 
    x: 0,
    transition: { duration: 0.3 },
  },
  exit: { 
    opacity: 0, 
    y: -20,
    transition: { duration: 0.2 },
  },
}

export function Toast({ message, onClose }: ToastProps) {
  return (
    <motion.div
      variants={toastVariants}
      initial="initial"
      animate="animate"
      exit="exit"
      className="fixed top-4 right-4 bg-white shadow-lg rounded-lg p-4"
    >
      {message}
    </motion.div>
  )
}
```

### Form Shake Animation
```typescript
// src/components/ui/input.tsx (enhanced)
import { motion } from 'framer-motion'

const shakeVariants = {
  shake: {
    x: [-10, 10, -10, 10, 0],
    transition: { duration: 0.4 },
  },
}

export function Input({ error, ...props }: InputProps) {
  return (
    <motion.input
      animate={error ? 'shake' : ''}
      variants={shakeVariants}
      className={cn(
        'border rounded-md px-3 py-2',
        error && 'border-error'
      )}
      {...props}
    />
  )
}
```

### Reduced Motion Support
```typescript
// src/utils/motion.ts
import { useReducedMotion } from 'framer-motion'

export function useAnimationConfig() {
  const shouldReduceMotion = useReducedMotion()

  return {
    transition: shouldReduceMotion 
      ? { duration: 0 } 
      : { duration: 0.3 },
  }
}

// Usage
const { transition } = useAnimationConfig()

<motion.div
  initial={{ opacity: 0 }}
  animate={{ opacity: 1 }}
  transition={transition}
>
  Content
</motion.div>
```

## Notes

- Use `transform` and `opacity` for GPU acceleration
- Avoid animating `width`, `height`, `top`, `left` (causes layout)
- Test with Chrome DevTools Performance tab
- Ensure animations don't block user interactions
- Use `AnimatePresence` for exit animations
- Consider adding spring animations for more natural feel
- Test on mobile devices for touch responsiveness
