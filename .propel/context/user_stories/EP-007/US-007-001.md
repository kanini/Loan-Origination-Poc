# User Story: Excel Export with SheetJS Multi-Sheet Workbook

**Epic:** EP-007 - Export Functionality & Multi-Format Support  
**Story ID:** US-007-001  
**Priority:** P0 - Critical  
**Estimate:** 8 Story Points (3-4 days)

---

## User Story

**As a** Financial Analyst  
**I want** to export extracted entity data to Excel with formatted multi-sheet workbooks  
**So that** I can analyze and share data in a familiar spreadsheet format with proper formatting

---

## Acceptance Criteria

### AC1: Export Button and Dropdown
**Given** I am on the Results page  
**When** I view the export options  
**Then**:
- "Export" dropdown button in top-right of entities panel
- Dropdown shows 4 options: "Download as JSON", "Download as CSV", "Download as Excel", "Generate PDF Report"
- Excel option has spreadsheet icon
- Dropdown is keyboard accessible
- Button shows loading spinner during export

### AC2: Client-Side Excel Generation
**Given** I click "Download as Excel"  
**When** the export is triggered  
**Then**:
- Excel file generated client-side using SheetJS library
- No server round-trip required
- Generation completes in <3 seconds for typical dataset
- Loading spinner shows on export button
- Button is disabled during generation

### AC3: Multi-Sheet Workbook Structure
**Given** Excel file is generated  
**When** I open the downloaded file  
**Then**:
- Workbook contains 6 worksheets:
  1. "Overview" (summary sheet)
  2. "Lender" (Lender Information entities)
  3. "Borrower" (Borrower Details entities)
  4. "Loan Terms" (Loan Terms entities)
  5. "Location" (Location Data entities)
  6. "Person" (Person Information entities)
- Sheets appear in this order (Overview first)
- Each sheet has descriptive tab name

### AC4: Overview Worksheet Content
**Given** I view the Overview worksheet  
**When** I examine the content  
**Then**:
- Row 1: "Document Summary" (bold, size 14)
- Row 2: Empty
- Row 3: "Filename:" | [document_name.pdf]
- Row 4: "Upload Date:" | [MMM DD, YYYY HH:MM]
- Row 5: "Model Used:" | [OpenAI GPT / Google Gemini]
- Row 6: "Processing Time:" | [X seconds]
- Row 7: "Total Entities Extracted:" | [count]
- Row 8: Empty
- Row 9: "Category Breakdown" (bold)
- Row 10+: Category name | Entity count for each category

### AC5: Entity Worksheet Structure
**Given** I view an entity category worksheet (e.g., "Lender")  
**When** I examine the structure  
**Then**:
- Row 1: Header row with columns "Field Name" | "Extracted Value"
- Header row has: Bold font, Blue background (#3B82F6), White text, Frozen pane
- Rows 2+: Entity data with field name in column A, value in column B
- Columns auto-sized to fit content width
- Empty categories show headers only (no data rows)

### AC6: Data Formatting
**Given** entity values are displayed in Excel  
**When** I view different data types  
**Then**:
- Currency values formatted as `$#,##0.00` (e.g., "$250,000.00")
- Percentage values formatted as `0.00%` (e.g., "5.25%")
- Dates formatted as `MMM DD, YYYY` (e.g., "Jan 15, 2026")
- Text values left-aligned
- Number values right-aligned
- Empty values show as empty cells (not "--")

### AC7: Filename Pattern
**Given** Excel file is downloaded  
**When** I check the filename  
**Then**:
- Filename follows pattern: `{document_name}_entities_{YYYY-MM-DD}.xlsx`
- Example: `loan_application_entities_2026-01-15.xlsx`
- Special characters in document name are sanitized (replaced with underscores)
- Filename is unique (no overwrites without confirmation)

### AC8: Edge Case Handling
**Given** various data scenarios  
**When** Excel is generated  
**Then**:
- Empty categories produce worksheet with headers only
- Special characters in field values are properly escaped
- Very long field names are truncated with ellipsis
- Null/undefined values show as empty cells
- Large datasets (>1000 rows) generate successfully
- Unicode characters (é, ñ, 中) display correctly

### AC9: Success Feedback
**Given** Excel export completes  
**When** the file is downloaded  
**Then**:
- Success toast notification: "Excel file downloaded successfully"
- Export button returns to normal state (no spinner)
- File downloads to browser's default download location
- User can immediately open the file

### AC10: Error Handling
**Given** Excel export fails  
**When** an error occurs  
**Then**:
- Error toast notification: "Export failed. Please try a different format or reduce data size"
- Export button returns to normal state
- User can retry export
- Error is logged for debugging

### Excel Export Implementation
```typescript
// src/utils/excelExport.ts
import * as XLSX from 'xlsx'
import { format } from 'date-fns'

interface EntityData {
  category: string
  fields: Array<{ name: string; value: any }>
}

interface DocumentMetadata {
  filename: string
  uploadDate: Date
  model: string
  processingTime: number
  totalEntities: number
}

export function generateExcelWorkbook(
  entities: EntityData[],
  metadata: DocumentMetadata
): XLSX.WorkBook {
  const workbook = XLSX.utils.book_new()

  // Create Overview worksheet
  const overviewData = [
    ['Document Summary'],
    [],
    ['Filename:', metadata.filename],
    ['Upload Date:', format(metadata.uploadDate, 'MMM dd, yyyy HH:mm')],
    ['Model Used:', metadata.model],
    ['Processing Time:', `${metadata.processingTime} seconds`],
    ['Total Entities Extracted:', metadata.totalEntities],
    [],
    ['Category Breakdown'],
    ...entities.map(cat => [cat.category, cat.fields.length]),
  ]

  const overviewSheet = XLSX.utils.aoa_to_sheet(overviewData)
  
  // Style Overview sheet
  overviewSheet['A1'].s = { font: { bold: true, sz: 14 } }
  overviewSheet['A9'].s = { font: { bold: true } }
  
  XLSX.utils.book_append_sheet(workbook, overviewSheet, 'Overview')

  // Create entity category worksheets
  entities.forEach(category => {
    const sheetData = [
      ['Field Name', 'Extracted Value'],
      ...category.fields.map(field => [field.name, formatValue(field.value)]),
    ]

    const sheet = XLSX.utils.aoa_to_sheet(sheetData)

    // Style header row
    sheet['A1'].s = {
      font: { bold: true, color: { rgb: 'FFFFFF' } },
      fill: { fgColor: { rgb: '3B82F6' } },
    }
    sheet['B1'].s = {
      font: { bold: true, color: { rgb: 'FFFFFF' } },
      fill: { fgColor: { rgb: '3B82F6' } },
    }

    // Freeze header row
    sheet['!freeze'] = { xSplit: 0, ySplit: 1 }

    // Auto-size columns
    const maxWidths = [
      Math.max(...category.fields.map(f => f.name.length), 10),
      Math.max(...category.fields.map(f => String(f.value).length), 15),
    ]
    sheet['!cols'] = maxWidths.map(w => ({ wch: Math.min(w + 2, 50) }))

    // Apply number formatting
    category.fields.forEach((field, idx) => {
      const cellRef = `B${idx + 2}`
      if (isCurrency(field.value)) {
        sheet[cellRef].z = '$#,##0.00'
      } else if (isPercentage(field.value)) {
        sheet[cellRef].z = '0.00%'
      }
    })

    XLSX.utils.book_append_sheet(workbook, sheet, sanitizeSheetName(category.category))
  })

  return workbook
}

export function downloadExcelFile(workbook: XLSX.WorkBook, filename: string) {
  const excelBuffer = XLSX.write(workbook, { bookType: 'xlsx', type: 'array' })
  const blob = new Blob([excelBuffer], { type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet' })
  
  const url = URL.createObjectURL(blob)
  const link = document.createElement('a')
  link.href = url
  link.download = `${sanitizeFilename(filename)}_entities_${format(new Date(), 'yyyy-MM-dd')}.xlsx`
  link.click()
  URL.revokeObjectURL(url)
}

function formatValue(value: any): any {
  if (value === null || value === undefined) return ''
  if (typeof value === 'object') return JSON.stringify(value)
  return value
}

function isCurrency(value: any): boolean {
  return typeof value === 'string' && /^\$[\d,]+\.?\d*$/.test(value)
}

function isPercentage(value: any): boolean {
  return typeof value === 'string' && /^\d+\.?\d*%$/.test(value)
}

function sanitizeSheetName(name: string): string {
  // Excel sheet names max 31 chars, no special chars
  return name.replace(/[:\\\/\?\*\[\]]/g, '_').slice(0, 31)
}

function sanitizeFilename(name: string): string {
  return name.replace(/[^a-zA-Z0-9_-]/g, '_')
}
```

### Component Integration
```typescript
// src/components/results/ExportDropdown.tsx
import { useState } from 'react'
import { Download, FileSpreadsheet } from 'lucide-react'
import { Button } from '@/components/ui/button'
import { DropdownMenu } from '@/components/ui/dropdown-menu'
import { useToast } from '@/hooks/useToast'
import { generateExcelWorkbook, downloadExcelFile } from '@/utils/excelExport'

export function ExportDropdown({ documentId, entities, metadata }) {
  const [isExporting, setIsExporting] = useState(false)
  const { toast } = useToast()

  const handleExcelExport = async () => {
    try {
      setIsExporting(true)
      
      const workbook = generateExcelWorkbook(entities, metadata)
      downloadExcelFile(workbook, metadata.filename)
      
      toast({
        title: 'Success',
        description: 'Excel file downloaded successfully',
        variant: 'success',
      })
    } catch (error) {
      toast({
        title: 'Export Failed',
        description: 'Please try a different format or reduce data size',
        variant: 'error',
      })
    } finally {
      setIsExporting(false)
    }
  }

  return (
    <DropdownMenu>
      <DropdownMenu.Trigger asChild>
        <Button disabled={isExporting}>
          {isExporting ? <Loader2 className="h-4 w-4 animate-spin" /> : <Download className="h-4 w-4" />}
          Export
        </Button>
      </DropdownMenu.Trigger>
      
      <DropdownMenu.Content>
        <DropdownMenu.Item onClick={handleExcelExport}>
          <FileSpreadsheet className="h-4 w-4 mr-2" />
          Download as Excel
        </DropdownMenu.Item>
        {/* Other export options */}
      </DropdownMenu.Content>
    </DropdownMenu>
  )
}
```
