# User Story: React Router Configuration with Lazy-Loaded Routes

**Epic:** EP-002 - Navigation & Information Architecture  
**Story ID:** US-002-001  
**Priority:** P0 - Critical  
**Estimate:** 5 Story Points (2 days)

---

## User Story

**As a** Frontend Developer  
**I want** to configure React Router v6 with lazy-loaded routes for Dashboard, Upload, Processing, Results, and History pages  
**So that** users can navigate between pages with client-side routing and deep linking support

---

## Acceptance Criteria

### AC1: React Router Installation and Setup
**Given** the React project is set up  
**When** I install and configure React Router v6  
**Then**:
- React Router DOM v6+ is installed as a dependency
- BrowserRouter wraps the entire application
- Routes are defined for all 5 main pages
- 404 Not Found route handles invalid URLs
- Base path is configured correctly

### AC2: Route Definitions
**Given** React Router is configured  
**When** I define application routes  
**Then** the following routes are available:
- `/` → Dashboard page
- `/upload` → Document Upload page
- `/processing/:documentId` → Processing Status page
- `/results/:documentId` → Results/Output Display page
- `/history` → Document History page
- `*` → 404 Not Found page

### AC3: Lazy-Loaded Route Components
**Given** routes are defined  
**When** I implement lazy loading  
**Then**:
- Each page component is loaded with React.lazy()
- Suspense boundaries wrap each route with skeleton fallback
- Route chunks are code-split into separate bundles
- Navigation triggers chunk loading on-demand
- Loading state shows during chunk download

### AC4: Deep Linking Support
**Given** routes are configured  
**When** users access URLs directly  
**Then**:
- Direct URL access loads the correct page (e.g., `/results/123`)
- Browser back/forward buttons work correctly
- Page refresh maintains current route
- URL parameters are accessible in components (documentId)
- Query parameters are preserved during navigation

### AC5: Navigation Guards (Future)
**Given** routes are defined  
**When** I add navigation logic  
**Then**:
- Placeholder for authentication guard (Phase 2)
- Redirect logic for protected routes (Phase 2)
- Route transition animations work smoothly
- Navigation state is accessible in components

### AC6: Route-Based Page Titles
**Given** users navigate between pages  
**When** the route changes  
**Then**:
- Browser tab title updates to reflect current page
- Page titles follow pattern: "Page Name - GenAI PDF Extractor"
- Document-specific pages show document name in title

## Technical Details

### Installation
```bash
npm install react-router-dom@^6.21.0
```

### Router Configuration
```typescript
// src/App.tsx
import { lazy, Suspense } from 'react'
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom'
import { PageSkeleton } from '@/components/common/PageSkeleton'
import { ErrorBoundary } from '@/components/common/ErrorBoundary'
import { PageLayout } from '@/components/layout/PageLayout'

// Lazy-loaded page components
const Dashboard = lazy(() => import('@/pages/Dashboard'))
const Upload = lazy(() => import('@/pages/Upload'))
const Processing = lazy(() => import('@/pages/Processing'))
const Results = lazy(() => import('@/pages/Results'))
const History = lazy(() => import('@/pages/History'))
const NotFound = lazy(() => import('@/pages/NotFound'))

function App() {
  return (
    <BrowserRouter>
      <ErrorBoundary>
        <PageLayout>
          <Suspense fallback={<PageSkeleton />}>
            <Routes>
              <Route path="/" element={<Dashboard />} />
              <Route path="/upload" element={<Upload />} />
              <Route path="/processing/:documentId" element={<Processing />} />
              <Route path="/results/:documentId" element={<Results />} />
              <Route path="/history" element={<History />} />
              <Route path="/404" element={<NotFound />} />
              <Route path="*" element={<Navigate to="/404" replace />} />
            </Routes>
          </Suspense>
        </PageLayout>
      </ErrorBoundary>
    </BrowserRouter>
  )
}

export default App
```

### Page Title Hook
```typescript
// src/hooks/usePageTitle.ts
import { useEffect } from 'react'

export function usePageTitle(title: string) {
  useEffect(() => {
    const previousTitle = document.title
    document.title = `${title} - GenAI PDF Extractor`
    
    return () => {
      document.title = previousTitle
    }
  }, [title])
}

// Usage in page component
import { usePageTitle } from '@/hooks/usePageTitle'

export default function Dashboard() {
  usePageTitle('Dashboard')
  
  return <div>Dashboard content</div>
}
```

### Navigation Helper
```typescript
// src/utils/navigation.ts
import { useNavigate, useParams, useSearchParams } from 'react-router-dom'

export function useAppNavigation() {
  const navigate = useNavigate()
  
  return {
    goToDashboard: () => navigate('/'),
    goToUpload: () => navigate('/upload'),
    goToProcessing: (documentId: string) => navigate(`/processing/${documentId}`),
    goToResults: (documentId: string) => navigate(`/results/${documentId}`),
    goToHistory: () => navigate('/history'),
    goBack: () => navigate(-1),
  }
}
```

### Route Parameters Access
```typescript
// src/pages/Results.tsx
import { useParams } from 'react-router-dom'

export default function Results() {
  const { documentId } = useParams<{ documentId: string }>()
  
  // Use documentId to fetch document data
  return <div>Results for document {documentId}</div>
}
```
