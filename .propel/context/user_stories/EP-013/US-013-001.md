# User Story: React Query Setup and Configuration

**Epic:** EP-013 - Backend API Integration & State Management  
**Story ID:** US-013-001  
**Priority:** P0 - Critical  
**Estimate:** 3 Story Points (1 day)

---

## User Story

**As a** Frontend Developer  
**I want** to configure React Query (TanStack Query) for server state management  
**So that** I can efficiently fetch, cache, and synchronize data from the Django backend with automatic background refetching

---

## Acceptance Criteria

### AC1: React Query Installation
**Given** the React project is set up  
**When** I install React Query  
**Then**:
- @tanstack/react-query v5.0+ installed as dependency
- @tanstack/react-query-devtools installed as dev dependency
- React Query DevTools available in development mode
- No version conflicts with other dependencies

### AC2: QueryClient Configuration
**Given** React Query is installed  
**When** I configure the QueryClient  
**Then**:
- QueryClient created with custom default options
- staleTime set to 5 minutes (300000ms)
- cacheTime set to 10 minutes (600000ms)
- refetchOnWindowFocus enabled
- refetchOnReconnect enabled
- retry logic configured (3 retries with exponential backoff)
- Error handling configured globally

### AC3: QueryClientProvider Setup
**Given** QueryClient is configured  
**When** I wrap the application  
**Then**:
- QueryClientProvider wraps entire app at root level
- QueryClient instance passed to provider
- All child components have access to React Query hooks
- Provider is placed inside Router but outside Suspense boundaries

### AC4: React Query DevTools
**Given** the application is in development mode  
**When** I run the dev server  
**Then**:
- React Query DevTools panel available in browser
- DevTools show all active queries and mutations
- Query cache inspection available
- Query invalidation and refetching can be triggered manually
- DevTools only included in development build (not production)

### AC5: Query Keys Convention
**Given** I create queries throughout the application  
**When** I define query keys  
**Then**:
- Query keys follow consistent naming convention
- Array format used for all keys: `['resource', id, filters]`
- Examples:
  - `['documents']` - all documents
  - `['documents', documentId]` - specific document
  - `['documents', 'recent']` - recent documents
  - `['extraction-results', documentId]` - extraction results
  - `['processing-status', documentId]` - processing status
- Query keys documented in central location

### AC6: Error Handling
**Given** API requests fail  
**When** errors occur  
**Then**:
- Global error handler catches all query errors
- Network errors retry automatically (3 attempts)
- 4xx errors don't retry (client errors)
- 5xx errors retry with exponential backoff
- Error messages are user-friendly
- Errors are logged for debugging

### AC7: Background Refetching
**Given** queries are cached  
**When** user returns to the application  
**Then**:
- Stale queries refetch in background on window focus
- Queries refetch on network reconnection
- Fresh data replaces stale data automatically
- UI shows loading indicators during refetch
- Cached data displayed while refetching

### AC8: Optimistic Updates
**Given** mutations are performed  
**When** I update data  
**Then**:
- UI updates immediately (optimistic update)
- Mutation sent to server in background
- On success: optimistic update confirmed
- On error: optimistic update rolled back
- Related queries invalidated and refetched

## Technical Details

### Installation
```bash
npm install @tanstack/react-query@^5.17.0
npm install --save-dev @tanstack/react-query-devtools@^5.17.0
```

### QueryClient Configuration
```typescript
// src/lib/queryClient.ts
import { QueryClient } from '@tanstack/react-query'

export const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 5 * 60 * 1000, // 5 minutes
      cacheTime: 10 * 60 * 1000, // 10 minutes
      refetchOnWindowFocus: true,
      refetchOnReconnect: true,
      retry: (failureCount, error: any) => {
        // Don't retry on 4xx errors (client errors)
        if (error?.response?.status >= 400 && error?.response?.status < 500) {
          return false
        }
        // Retry up to 3 times for other errors
        return failureCount < 3
      },
      retryDelay: (attemptIndex) => {
        // Exponential backoff: 1s, 2s, 4s
        return Math.min(1000 * 2 ** attemptIndex, 30000)
      },
    },
    mutations: {
      retry: false, // Don't retry mutations by default
      onError: (error: any) => {
        console.error('Mutation error:', error)
        // Global error handling for mutations
      },
    },
  },
})
```

### Provider Setup
```typescript
// src/App.tsx
import { QueryClientProvider } from '@tanstack/react-query'
import { ReactQueryDevtools } from '@tanstack/react-query-devtools'
import { queryClient } from '@/lib/queryClient'

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <BrowserRouter>
        {/* App content */}
      </BrowserRouter>
      
      {/* DevTools only in development */}
      {process.env.NODE_ENV === 'development' && (
        <ReactQueryDevtools initialIsOpen={false} position="bottom-right" />
      )}
    </QueryClientProvider>
  )
}
```

### Query Keys Convention
```typescript
// src/lib/queryKeys.ts
export const queryKeys = {
  documents: {
    all: ['documents'] as const,
    lists: () => [...queryKeys.documents.all, 'list'] as const,
    list: (filters: any) => [...queryKeys.documents.lists(), filters] as const,
    details: () => [...queryKeys.documents.all, 'detail'] as const,
    detail: (id: string) => [...queryKeys.documents.details(), id] as const,
    recent: () => [...queryKeys.documents.all, 'recent'] as const,
    statistics: () => [...queryKeys.documents.all, 'statistics'] as const,
  },
  extractionResults: {
    all: ['extraction-results'] as const,
    detail: (documentId: string) => [...queryKeys.extractionResults.all, documentId] as const,
  },
  processingStatus: {
    all: ['processing-status'] as const,
    detail: (documentId: string) => [...queryKeys.processingStatus.all, documentId] as const,
  },
}
```

### Example Query Usage
```typescript
// src/hooks/useDocuments.ts
import { useQuery } from '@tanstack/react-query'
import { queryKeys } from '@/lib/queryKeys'
import { fetchDocuments } from '@/api/documents'

export function useDocuments(filters?: any) {
  return useQuery({
    queryKey: queryKeys.documents.list(filters),
    queryFn: () => fetchDocuments(filters),
  })
}

export function useDocument(documentId: string) {
  return useQuery({
    queryKey: queryKeys.documents.detail(documentId),
    queryFn: () => fetchDocument(documentId),
    enabled: !!documentId, // Only run if documentId exists
  })
}
```

### Example Mutation Usage
```typescript
// src/hooks/useUpdateEntity.ts
import { useMutation, useQueryClient } from '@tanstack/react-query'
import { queryKeys } from '@/lib/queryKeys'
import { updateEntity } from '@/api/entities'

export function useUpdateEntity(documentId: string) {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: updateEntity,
    onMutate: async (newData) => {
      // Cancel outgoing refetches
      await queryClient.cancelQueries({ 
        queryKey: queryKeys.extractionResults.detail(documentId) 
      })

      // Snapshot previous value
      const previousData = queryClient.getQueryData(
        queryKeys.extractionResults.detail(documentId)
      )

      // Optimistically update
      queryClient.setQueryData(
        queryKeys.extractionResults.detail(documentId),
        (old: any) => ({ ...old, ...newData })
      )

      return { previousData }
    },
    onError: (err, newData, context) => {
      // Rollback on error
      queryClient.setQueryData(
        queryKeys.extractionResults.detail(documentId),
        context?.previousData
      )
    },
    onSuccess: () => {
      // Invalidate and refetch
      queryClient.invalidateQueries({ 
        queryKey: queryKeys.extractionResults.detail(documentId) 
      })
    },
  })
}
```
