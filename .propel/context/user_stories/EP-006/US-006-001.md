# User Story: Results Display with Card and Table Views

**Epic:** EP-006 - Results Display & Entity Visualization  
**Story ID:** US-006-001  
**Priority:** P0 - Critical  
**Estimate:** 13 Story Points (5-6 days)

---

## User Story

**As a** Financial Analyst  
**I want** to view extracted entities in both card and table layouts with inline editing  
**So that** I can review and correct entity data in my preferred format efficiently

---

## Acceptance Criteria

### AC1: Results Page Layout
**Given** I navigate to the Results page  
**When** the page loads  
**Then**:
- Document info header displays: filename (H1), upload date, model used, processing time
- Action buttons in header: Edit, Export, Delete
- Side-by-side layout on desktop (≥1024px): 60% entities / 40% PDF viewer
- Tabbed interface on tablet (768px-1023px): Entities tab, PDF tab
- Stacked layout on mobile (<768px): Entities section, collapsible PDF section
- View toggle switch: "Card View" | "Table View" (top-left of entities panel)
- Export dropdown button (top-right of entities panel)

### AC2: Entity Organization by Category
**Given** extraction results are displayed  
**When** I view the entities  
**Then**:
- Entities organized into 5 categories:
  - Lender Information
  - Borrower Details
  - Loan Terms
  - Location Data
  - Person Information
- Each category displayed as separate card (card view) or table (table view)
- Categories shown in consistent order
- Empty categories display with "--" placeholders

### AC3: Card View Display
**Given** I select Card View  
**When** I view the entities  
**Then**:
- Each category in separate card component
- Card has header with category name (H3)
- Field label and extracted value displayed as key-value pairs
- Labels use Small typography (14px/400), values use Body (16px/400)
- Confidence scores shown as badges when available (e.g., "95%")
- Empty fields show "--" with muted gray color
- Cards have white background, shadow-sm, 24px padding
- Cards stack vertically with 16px spacing

### AC4: Table View Display
**Given** I select Table View  
**When** I view the entities  
**Then**:
- Each category rendered as dedicated table
- Table columns: "Field Name" | "Extracted Value"
- Table headers are sticky when scrolling
- Row striping (alternating background colors)
- Sortable columns (click header to sort ascending/descending)
- Quick-search filter input above each table
- Tables have proper semantic HTML (`<thead>`, `<tbody>`, `<th scope>`)
- Horizontal scrolling on mobile, full-width on desktop

### AC5: View Toggle Functionality
**Given** I am viewing entities  
**When** I click the view toggle switch  
**Then**:
- View switches between Card and Table instantly (<100ms)
- All data and edit state preserved during toggle
- Scroll position maintained where possible
- Toggle state persisted in localStorage
- Toggle is keyboard accessible (Tab, Space, Enter)

### AC6: Inline Editing (Card View)
**Given** I am in Card View  
**When** I click on a field value  
**Then**:
- Field enters edit mode with text input
- Current value pre-filled in input
- Input has focus with cursor at end
- Pressing Enter or clicking outside saves changes
- Pressing Esc cancels edit and reverts value
- Validation errors show inline with red border
- Success toast shows: "Field updated successfully"

### AC7: Inline Editing (Table View)
**Given** I am in Table View  
**When** I click on a cell in "Extracted Value" column  
**Then**:
- Cell enters edit mode with text input
- Current value pre-filled in input
- Input has focus with cursor at end
- Pressing Enter or Tab saves and moves to next cell
- Pressing Esc cancels edit
- Validation errors show inline
- Optimistic update shows immediately

### AC8: Sortable Tables
**Given** I am in Table View  
**When** I click a table column header  
**Then**:
- Table rows sort by that column (ascending)
- Clicking again sorts descending
- Sort indicator icon shows direction (↑ or ↓)
- Sorting is instant (<100ms)
- Sort state maintained during view toggle

### AC9: Quick-Search Filter
**Given** I am in Table View  
**When** I type in the quick-search input  
**Then**:
- Table rows filter in real-time (debounced 300ms)
- Matching rows show, non-matching rows hide
- Search is case-insensitive
- Search matches both field names and values
- Clear button (X) clears search
- "No results found" message when no matches

### AC10: Confidence Scores
**Given** LLM provides confidence scores  
**When** I view entity fields  
**Then**:
- Confidence score badge displayed next to value
- High confidence (≥90%): Green badge
- Medium confidence (70-89%): Amber badge
- Low confidence (<70%): Red badge
- Badge shows percentage (e.g., "85%")
- Tooltip explains confidence meaning

### AC11: Empty Field Handling
**Given** an entity field has no extracted value  
**When** I view that field  
**Then**:
- Placeholder "--" displayed in muted gray
- Field is still editable (click to add value)
- Empty fields don't break layout
- Tooltip shows "No value extracted"

### AC12: Responsive Behavior
**Given** I view Results on different devices  
**When** I resize the viewport  
**Then**:
- Desktop (≥1024px): Side-by-side entities and PDF
- Tablet (768px-1023px): Tabbed interface
- Mobile (<768px): Stacked layout, card view only
- Tables scroll horizontally on mobile
- All touch targets ≥44×44px on mobile

## Technical Details

### Component Structure
```typescript
// src/pages/Results.tsx
import { useState } from 'react'
import { useParams } from 'react-router-dom'
import { useQuery } from '@tanstack/react-query'
import { LayoutGrid, Table as TableIcon } from 'lucide-react'
import { Toggle } from '@/components/ui/toggle'
import { EntityCardView } from '@/components/results/EntityCardView'
import { EntityTableView } from '@/components/results/EntityTableView'
import { PDFViewer } from '@/components/results/PDFViewer'
import { fetchExtractionResults } from '@/api/documents'

type ViewMode = 'card' | 'table'

export default function Results() {
  const { documentId } = useParams()
  const [viewMode, setViewMode] = useState<ViewMode>(() => {
    return (localStorage.getItem('resultsViewMode') as ViewMode) || 'card'
  })

  const { data, isLoading } = useQuery({
    queryKey: ['extraction-results', documentId],
    queryFn: () => fetchExtractionResults(documentId!),
  })

  const handleViewModeChange = (mode: ViewMode) => {
    setViewMode(mode)
    localStorage.setItem('resultsViewMode', mode)
  }

  if (isLoading) return <ResultsSkeleton />

  return (
    <div className="max-w-screen-2xl mx-auto px-4 py-8">
      {/* Document Header */}
      <DocumentHeader document={data.document} />

      {/* Main Content */}
      <div className="grid grid-cols-1 lg:grid-cols-5 gap-6 mt-6">
        {/* Entities Panel (60%) */}
        <div className="lg:col-span-3 space-y-4">
          {/* View Toggle */}
          <div className="flex items-center justify-between">
            <div className="flex items-center gap-2">
              <Toggle
                pressed={viewMode === 'card'}
                onPressedChange={() => handleViewModeChange('card')}
                aria-label="Card view"
              >
                <LayoutGrid className="h-4 w-4 mr-2" />
                Card View
              </Toggle>
              <Toggle
                pressed={viewMode === 'table'}
                onPressedChange={() => handleViewModeChange('table')}
                aria-label="Table view"
              >
                <TableIcon className="h-4 w-4 mr-2" />
                Table View
              </Toggle>
            </div>

            <ExportDropdown documentId={documentId} />
          </div>

          {/* Entity Display */}
          {viewMode === 'card' ? (
            <EntityCardView entities={data.entities} />
          ) : (
            <EntityTableView entities={data.entities} />
          )}
        </div>

        {/* PDF Viewer Panel (40%) */}
        <div className="lg:col-span-2">
          <PDFViewer pdfUrl={data.document.pdfUrl} />
        </div>
      </div>
    </div>
  )
}
```
