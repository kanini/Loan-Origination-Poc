# User Story: Responsive Layouts for Desktop, Tablet, and Mobile

**Epic:** EP-012 - Responsive Design & Cross-Device Support  
**Story ID:** US-012-001  
**Priority:** P0 - Critical  
**Estimate:** 8 Story Points (3-4 days)

---

## User Story

**As a** Financial Analyst using various devices  
**I want** the application to adapt seamlessly to different screen sizes  
**So that** I can work efficiently on desktop, tablet, or mobile devices

---

## Acceptance Criteria

### AC1: Responsive Breakpoints
**Given** the application is configured  
**When** I define breakpoints  
**Then**:
- Mobile: <768px (default, no prefix)
- Tablet: 768px-1023px (`md:` prefix)
- Desktop: ≥1024px (`lg:` prefix)
- Breakpoints configured in TailwindCSS
- Consistent across all components

### AC2: Desktop Layout (≥1024px)
**Given** I view the application on desktop  
**When** the viewport is ≥1024px  
**Then**:
- Multi-column layouts utilized
- Dashboard: 3-column grid
- Results page: 60% entities / 40% PDF side-by-side
- Navigation: Full horizontal menu
- Tables: Full width with all columns visible
- Maximum content width: 1280px (centered)
- Generous whitespace and padding

### AC3: Tablet Layout (768px-1023px)
**Given** I view the application on tablet  
**When** the viewport is 768px-1023px  
**Then**:
- Adaptive 2-column layouts
- Dashboard: 2-column grid
- Results page: Tabbed interface (Entities tab, PDF tab)
- Navigation: Full horizontal menu or collapsible
- Tables: Horizontal scroll if needed
- Reduced padding compared to desktop
- Touch-friendly controls (≥44×44px)

### AC4: Mobile Layout (<768px)
**Given** I view the application on mobile  
**When** the viewport is <768px  
**Then**:
- Single-column stacked layouts
- Dashboard: Stacked cards
- Results page: Stacked entities, collapsible PDF
- Navigation: Hamburger menu
- Tables: Horizontal scroll or card view
- Full-width components
- Touch-optimized controls (≥44×44px)
- Minimal padding (16px)

### AC5: Dashboard Responsiveness
**Given** I view the Dashboard  
**When** I resize the viewport  
**Then**:
- Desktop: 3-column grid (recent docs + stats)
- Tablet: 2-column grid
- Mobile: Single column, stacked
- Cards resize appropriately
- Images/thumbnails scale
- Text remains readable
- No horizontal scrolling

### AC6: Upload Page Responsiveness
**Given** I view the Upload page  
**When** I resize the viewport  
**Then**:
- Desktop: Centered form (max-width 800px)
- Tablet: Slightly narrower (max-width 700px)
- Mobile: Full width with padding
- Drag-drop zone scales appropriately
- Model cards stack on mobile
- Buttons full-width on mobile
- Step indicator adapts

### AC7: Results Page Responsiveness
**Given** I view the Results page  
**When** I resize the viewport  
**Then**:
- Desktop: Side-by-side (60% entities, 40% PDF)
- Tablet: Tabbed interface (Entities tab, PDF tab)
- Mobile: Stacked (Entities, collapsible PDF)
- Card view on mobile (no table view)
- PDF viewer full-width on mobile
- Export button accessible on all sizes

### AC8: History Page Responsiveness
**Given** I view the History page  
**When** I resize the viewport  
**Then**:
- Desktop: 3-4 column grid or full-width table
- Tablet: 2 column grid or scrollable table
- Mobile: Single column cards (no grid)
- Search bar full-width on mobile
- Filters stack vertically on mobile
- Pagination controls adapt

### AC9: Images and Media
**Given** images and PDFs are displayed  
**When** I resize the viewport  
**Then**:
- Images scale proportionally
- PDFs fit container width
- Thumbnails resize appropriately
- No image overflow
- Aspect ratios maintained
- Lazy loading on mobile (performance)

### AC10: Text Readability
**Given** I view text content  
**When** I resize the viewport  
**Then**:
- Text remains readable on all sizes
- No horizontal scrolling required
- Line length optimal (45-75 characters)
- Font sizes scale appropriately
- Headings maintain hierarchy
- No text truncation unless intentional

## Technical Details

### TailwindCSS Responsive Utilities
```typescript
// Responsive Grid Example
<div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
  {/* Cards */}
</div>

// Responsive Padding
<div className="px-4 md:px-6 lg:px-8">
  {/* Content */}
</div>

// Responsive Text
<h1 className="text-2xl md:text-3xl lg:text-4xl">
  Title
</h1>

// Responsive Display
<div className="hidden lg:block">
  {/* Desktop only */}
</div>
<div className="block lg:hidden">
  {/* Mobile/Tablet only */}
</div>
```
