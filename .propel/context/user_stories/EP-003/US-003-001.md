## Story ID
   * ID Format: US-003-001

## Story Title
   * Unified Multi-File Upload Interface

## Description
  * As a Loan Processor, I want to upload multiple PDF documents (loan application and tax returns) through a single unified interface, so that I can quickly process dual-document extractions with minimal effort

## Acceptance Criteria
  * Given user navigates to Upload page, When the page renders, Then a single unified drag-and-drop zone displays:
    - Dashed border with visual feedback on drag-over
    - Click-to-browse file selection fallback
    - Support for up to 10 files per upload session
  * Given user selects files to upload, When files are added, Then validation enforces:
    - PDF files only (reject other formats with error message)
    - Maximum 10MB per file
    - Duplicate filename detection
    - Inline error messages immediately shown
  * Given files are validated, When files appear in the list, Then each file shows:
    - PDF icon, filename, file size
    - Individual remove button (X icon)
    - Clear visual indication of upload order
  * Given files are queued for upload, When user reviews upload form, Then model selection displays:
    - Radio buttons/cards for OpenAI GPT and Gemini
    - Model descriptions for each option
    - "Extract Entities" button disabled until file + model selected

## Edge Cases
   * What happens when non-PDF file is uploaded? Show clear error with accepted formats list.
   * How does system handle file exceeding 10MB? Show size limit error with actual file size.
   * What happens when duplicate filename is detected? Show warning with option to replace or rename.
   * How does system handle drag-drop of folder? Show error that folders are not supported.
