# User Story: Unified Multi-File Upload Interface

**Epic:** EP-003 - Document Upload & Multi-Document Processing  
**Story ID:** US-003-001  
**Priority:** P0 - Critical  
**Estimate:** 8 Story Points

---

## User Story

**As a** Loan Processor  
**I want** to upload multiple PDF documents (loan application and tax returns) through a single unified interface  
**So that** I can quickly process dual-document extractions with minimal effort

---

## Acceptance Criteria

### AC1: Drag-and-Drop Upload Zone
**Given** user navigates to Upload page  
**When** the page renders  
**Then** a single unified drag-and-drop zone displays:
- Dashed border with visual feedback on drag-over
- Click-to-browse file selection fallback
- Support for up to 10 files per upload session

### AC2: File Validation
**Given** user selects files to upload  
**When** files are added  
**Then** validation enforces:
- PDF files only (reject other formats with error message)
- Maximum 10MB per file
- Duplicate filename detection
- Inline error messages immediately shown

### AC3: File List Management
**Given** files are validated  
**When** files appear in the list  
**Then** each file shows:
- PDF icon, filename, file size
- Individual remove button (X icon)
- Clear visual indication of upload order

### AC4: LLM Model Selection
**Given** files are queued for upload  
**When** user reviews upload form  
**Then** model selection displays:
- Radio buttons/cards for OpenAI GPT and Gemini
- Model descriptions for each option
- "Extract Entities" button disabled until file + model selected
